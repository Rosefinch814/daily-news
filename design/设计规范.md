## 0. 设计基调（一句话）

**暖纸高端报刊**：做旧暖新闻纸 + 墨黑衬线正文 + Fraunces 展示衬线标题 + 朱红作唯一结构签名色。头版强主次、信息密度高、耐读，像一份属于自己的高端日报（参照 The Economist / 严肃 newsletter 的克制感）。**靠排版功力，不靠装饰小部件。**

## 0.1 v1 渲染范围（与产品 scope 对齐 · 落地必读）

本规范描述的是**完整产品**的视觉；**v1（精简验证版）只渲染其中一部分**：

- **✅ v1 渲染**：顶部红带 · 报头 · sticky 跳转条 · 头条精读（hero + 次条）· 速览。
- **⏭ M2 再加**：今日导读（`.lede-block`）· 关于本期（`.stat-strip`）· 反馈批注（每条 `.fb` 常驻 👍/👎 + 头条补充一句）· 整期留言（`.issue-note` + `.btn-record` + `.fb-recap`）· 归档导航（`.archive`）。对应跳转条目录项与锚点 `#lede`、`#feedback` 一并 M2 才出现。
- **🗑 已废弃（产品移除，勿做）**：「为什么给你」`.why` 块——头条判断块只保留「影响 · AI 分析」，`.judgement` 不再分栏（影响块占满整行）。

> 下面各组件描述全部保留（M2 仍照用）；落地 v1 时按上面取舍。产品 scope 权威：[docs/mvp](../docs/mvp/01-科技日报.md)、[里程碑 M1/M2](../docs/milestones/01-科技日报.md)。

## 1. 配色 Token

以 CSS 变量为唯一来源，组件内**禁止写裸 hex**，一律引用变量。

| 变量 | 值 | 用途 |
|---|---|---|
| `--paper` | `#F4EFE3` | 页面底色（做旧暖新闻纸）；正文永远压在它上 |
| `--paper-2` | `#EBE3D1` | 次级面板/hover/分隔（stat 条、速览 hover） |
| `--ink` | `#1A1714` | 正文与标题主色；重描边线 |
| `--ink-2` | `#3D372F` | 次级文字（hero 摘要、精读、影响正文、来源外文字） |
| `--ink-3` | `#5A5247` | 三级文字（速览正文） |
| `--muted` | `#8E836F` | 弱化（mono 元信息、编号、role 注） |
| `--rule` | `#D2C8B2` | 发丝线（区段线、卡片分隔） |
| `--rule-2` | `#B7AB90` | 稍重的边框（事实 tag 边；M2 组件边框/分隔）。速览列表分隔线用 `--rule` |
| `--seal` | `#A6342B` | **朱红 = 唯一结构签名色**（见 §1.1） |
| `--seal-deep` | `#8A2A22` | 朱红深色（备用：hover/按压） |
| `--seal-soft` | `#F0DCD5` | 朱红极浅底（选区、focus ring、软底） |
| `--ai-bg` | `#ECE7DA` | 「影响·AI 分析」块的异色底（与纸面区分） |

### 1.1 用色铁律
- **朱红 `--seal` 是唯一强调色，且用作"结构签名"**：顶部 6px 红带、区段标号小方块、kicker、首字下沉、链接下划线、「影响」块左边线、focus ring。**克制即高级——不要再引入第二强调色**（墨绿/蓝等第二色方案已在选型中淘汰）。
- **正文绝不压在彩色块上**：所有摘要/精读/正文一律 `--ink`/`--ink-2` 压 `--paper`。彩色只用于短标签、线条、标号。
- **事实 vs AI 判断必须视觉分区**（产品铁律，见 §9.2）。
- 对比度过 WCAG AA：正文 `--ink` on `--paper` ≈ 12:1；次级文字 ≥ 4.5:1。新增色必须复验。

## 2. 字体系统

三套字体各司其职，经 Google Fonts 引入，`display=swap`。

| 角色 | 字体 | 用在哪 |
|---|---|---|
| 展示衬线（Display） | **Fraunces**（opsz 光学尺寸；含 italic） | 刊名、所有大标题、阿拉伯数字/编号、抽栏引言（pullquote, italic）。**带个性，去"默认 AI 味"的关键。** |
| 正文/中文衬线 | **Noto Serif SC** | 正文、摘要、精读、中文标题字形 |
| 等宽/元信息（Mono） | **JetBrains Mono** | kicker、区段 role 注、来源、日期、期号、stat 数字、跳转条、`tnum` 表格数字 |

**关键技巧**：标题 `font-family:"Fraunces","Noto Serif SC",serif;` —— 中文走 Noto Serif SC，内嵌英文/数字（Rubin、AI、API、OpenAI）自动落到 Fraunces，立刻有个性。
**首字下沉**用 Noto Serif SC 900（Fraunces 无中文字形）。

## 3. 间距与栅格

- **内容最大宽** `--maxw: 1020px`；页面左右内边距 34px；底部 96px。
- **顶部全宽红带** 6px `--seal`（页面第一眼的签名）。
- **区段间距** 约 46px（`.section-label` 上外距）。
- **节奏**：模块之间用 1px 发丝线或 2px `--ink` 实线分层；hero 与次条之间、masthead 下、archive 上用 **2px `--ink`** 强线。
- **响应式断点**：
  - `≤820px`：次条双栏→单栏；masthead 竖排。（M2 的导读两栏、judgement 分栏同理塌为单栏。）
  - `≤520px`：速览两栏列表→单栏（`column-count:1`）。
- `html{scroll-padding-top:60px;}` 适配 sticky 跳转条。

## 4. 版面骨架（信息架构 · 竖版高屏效）

固定竖向顺序，从上到下浏览；每模块**各有分工、长得不一样**：

```
红带 ─ 报头(masthead：刊名 + 板块英文副标题 + 日期) ─ sticky 跳转条
└ 今日导读（综述 + 右侧"本期头条"目录，可点跳转）
└ 头条精读：头版 hero 主条（超大）+ 次条 2/3 小双栏退让
└ 速览（两栏简讯栏列表，条数自由流动）
└ 关于本期（一行 stat 条）
└ 反馈区（说明 + 文本框，无后端，复制喂下一期）
└ 归档导航（上一期 · 归档 · 下一期）
```
**主次原则**：每期只有一个 hero 主角（头版头条），其余一律退让。这是"重点突出、不平庸"的根本。

## 5. 组件规范

> 命名沿用 v1a 的 class，便于 Codex 对照。每个组件只描述样式要点。

- **顶部红带 `.redband`**：全宽 6px `--seal`。
- **报头 `.masthead`**：左 = 刊名 h1「Tourbillion News」(Fraunces 900，中缀 `.` 用 `--seal` 作签名色点睛) + 板块英文副标题 `.sub`（JetBrains Mono 大写 17px/.2em/500 `--seal`，如 "TECHNOLOGY"，须清晰可读、非小角标）；右 = 日期栏(Fraunces，仅日期·周几)。底部 2px `--ink` 线。**不显示期号、不放「全程中文」标语**（期号仅作内部/归档字段；全程中文是产品铁律 §3，不必在版面重申）。
- **跳转条 `.jump`**：`position:sticky;top:0`，半透明纸底 + `blur(8px)`，mono 小字大写；**滚动高亮当前区段**（`.active` 变 `--seal`，IntersectionObserver 实现，见 §6）。
- **区段标题 `.section-label`**：10px `--seal` 小方块 `.tab` + cn(中文) + role(英文 mono 注) + 占满的发丝线。
- **今日导读**：`.lede-block` = `grid 1fr 240px`。左 `.lede`（首字下沉 `--seal`）；右 `.lede-index`（"本期头条"序号列表，序号 Fraunces 13px `--muted`，hover 变 `--seal`，**默认无下划线**——保持安静）。
- **头版 hero `.hero`**：`.kicker`（栏目·动作 + `／头版头条`）→ 超大 `h2` → `.summary`（带 `事实` tag，**标题下唯一引导**，hero 内约 17.5px）→ 精读 `<details>` → judgement → 来源。底部 2px `--ink` 与次条分隔。**已删除旧斜体导语 `.standfirst`**（曾是标题下第二段引言，与摘要重复、且非 spec 字段；解读统一进「影响·AI 分析」块）。
- **次条 `.secondary > .sec`**：`grid 1fr 1fr`，左栏右边 1px 分隔线；标题 `.t` 22px，judgement 收成单列。
- **事实标 `.tag-fact`**：mono 9.5px，描边 `--rule-2`，文字"事实"。
- **精读 `<details>/summary`**：summary 为 mono 小字 + 16px `--seal` 短横 `.tw`（展开转 90°）；`.read-body` 展开淡入上移。
- **抽栏引言 `.pullquote`**：Fraunces italic 23px，左 3px `--seal` 竖线。
- **判断块 `.judgement`**：`grid 1.5fr 1fr`。
  - **影响 `.ai-impact`**：`--ai-bg` 底 + 左 3px `--seal` 线；标注 `.mk` = "影响 · AI 分析（非原文事实）"。
  - **为什么 `.why`**：左 1px `--rule-2` 细线；标注 `.mk` = "为什么给你"，命中关注点用 `<b>` 加重。
- **速览 `.brief-list`**：**两栏密排列表（报纸"简讯栏"）**——`column-count:2` + `column-gap:40px` + `column-rule:1px var(--rule)`；条目 `.brief` 用 `break-inside:avoid` 防跨栏断开、`border-top:1px var(--rule)` 作行分隔（首条去线）。**用 column 分栏而非 grid 格子**，条数随内容自由流动、不被栏数绑死，每条文字空间更大。每条紧凑高密度（`padding:9px 0`）：`.bno`(Fraunces 朱红编号) + 标题(600,15px) → 一句话摘要(`--ink-2`,13.5px,可写较实一句) → 来源(mono `--muted`)。`≤520px` 收为单栏。
- **关于本期 `.stat-strip`**：一行，左 3px `--seal` 边；标签 mono `--seal`，数字 `<b>` mono `--ink`，分隔点 `--rule-2`。
- **反馈批注 `.fb`（M2；入口形态＝常驻微缩图标）**：每条「来源·原文」行右侧常驻 `.fb-acts`——👍 `.fb-up` / 👎 `.fb-down` 极小细描边 SVG（**非 emoji**），默认 `--muted`，**选中态 `.is-on` ＝朱红填充**（`fill/stroke:var(--seal)`）。
  - **头条**：表态后（`.fb.reacted`）才显形 `.fb-note-wrap`＝单行 `.fb-note`（米白底 `--field#FBF8EF` + `--rule-2` 边，focus 改 `--seal` 边 + 3px `--seal-soft` ring）+ `.fb-note-submit`「记下」（朱红描边小按钮，hover 反白）。
  - **速览**（`.fb-mini`）：只 👍/👎，收在来源行同一行，**无补充输入**。
  - 写后淡出 `.fb-ack`「已记下 · 调整下一期」（mono `--seal`）。视觉/交互参考原型 [prototype/m2-反馈交互-科技日报.html](prototype/m2-反馈交互-科技日报.html)；交互定论见 [spec §14.1](../docs/spec/01-科技日报.md)。
- **整期留言 `.issue-note` + `.btn-record`（M2）**：报纸末尾多行 `textarea`（同米白底 + `--rule-2` 边 + focus ring）+ 朱红描边「记下本期」按钮（多行需显式提交）；上方 `.fb-recap` mono 小字汇总本期标记（朱红 `<b>`）。**反馈采集走 Supabase（anon key + RLS 仅 INSERT），不再是旧的"复制喂下一期"**。
- **归档 `nav.archive`**：mono 大写，上 2px `--ink` 线，三项分布两端与居中。
- **链接 `.ul`**：朱红"下划线划入"——`background` 线 `background-size:0→100% 1.5px`，`.3s` 缓动。

## 6. 交互与动效（克制·几处点睛）

零依赖、`prefers-reduced-motion:reduce` 时全部关闭（`*{animation:none;transition:none}` + `scroll-behavior:auto`）。

1. **链接下划线划入** `.ul`（0→100%，.3s `cubic-bezier(.2,.7,.2,1)`）。
2. **精读平滑展开** `.read-body` 淡入 + 上移 6px（.4s）；summary 短横展开转 90°。
3. **跳转条滚动高亮**：IntersectionObserver 观察 `#lede/#headlines/#briefs/#feedback`，命中区给对应 `.jump a` 加 `.active`（`rootMargin:'-55px 0px -65% 0px'`）。
4. **速览/文本框 hover/focus** 轻过渡。
- 所有动效 150–400ms；不做大幅位移、不阻塞阅读。

## 7. 可读性与无障碍

- 正文 ≥16px；行高 1.72–1.84。**hero 摘要铺满内容宽**（与下方「影响」块左右对齐，不留半幅空白）；精读/正文在各自容器内自然换行，次条摘要随栏宽。
- 对比度过 AA（§1.1）。
- **不用 emoji 当结构图标**：曾用的 📊/💡 已删，改排版化文字标注（既去 AI 味，又不依赖字体渲染）。
- **颜色不作唯一信息载体**：事实有"事实"字样、AI 判断有"影响 · AI 分析"字样 + 异底 + 左线三重区分。
- 链接可聚焦；focus 有可见态（textarea 的 ring；链接下划线）。

## 8. 全中文与文风（呼应产品铁律）

- **全程中文可读**：英文源内容融进中文，英文只出现在刊名副题/栏目代号/来源名等"装饰性元信息"。
- 文风：专业简报体，信息密度高、精炼。

## 9. 事实 / AI 判断的视觉契约（产品铁律 → 设计落地）

1. **事实**（摘要、精读）：常规墨色正文 + `事实` tag。忠于原文。
2. **AI 判断**（影响分析）：必须进 `.ai-impact` 块——异色底 `--ai-bg` + 左 `--seal` 线 + 明确标注"**影响 · AI 分析（非原文事实）**"。任何 AI 推断不得混入事实正文。
3. **为什么给你**：点明命中的关注点，属说明性，进 `.why` 块。

---

## 10. 扩展一致性（多板块/多期）

- 新增板块（如"财经"）**复用同一套 token、字体、组件与骨架**，只换内容与板块名；不另起视觉。
- 报头刊名「Tourbillion News」固定不变；**板块英文副标题随板块变**（Technology → Finance …），其余不动。
- 每期 HTML 复用同一组件结构与 class 命名；锚点固定 `#lede/#headlines/#briefs/#feedback` 与 `#hl1…`。
- 配色/字阶若需调整，**改 token 一处生效**，不在组件内散写。

## 12. 小红书图组卡片（竖版导出 · 新媒介）

> 把某一期日报导成一组竖版图（封面/头条/速览）发小红书。**尺寸 1080×1440**，是与网页日报**有意分叉**的新媒介，但基调仍是 V1a 报纸感。
> **和解结论**：卡片**完全复用 §1 基础 token，未引入任何新裸值/新色**（满足 [spec §6](../docs/spec/小红书图组导出.md) 的对齐要求）。
> 视觉基准原型：[prototype/xhs-图组-科技日报.html](prototype/xhs-图组-科技日报.html)（三种卡各一，文案已按草案区间忠实收敛演示）。导出行为、AI 收敛、字数契约属产品行为，权威在 [spec/小红书图组导出.md](../docs/spec/小红书图组导出.md)。

### 12.1 通用

- 卡片基座 `.card`：`1080×1440`，`--paper` 底，`overflow:hidden`（超高即拆页，不裁切正文——由分页逻辑保证不溢出）。
- 页眉 `.topbar`：mono 22px；左 = 卡别（`HEADLINE 1` / `BRIEFS`，`--seal` 700），右 = 日期（`--muted`）。封面不用 topbar。
- 页脚 `.foot`：绝对贴底，mono 20px；左「AI科技日报 · 日期」，右页码（`--seal` 700，如「头条 1 / 3」「速览 1 / 3」）。
- 字体沿用 §2 三套；正文/标题走 Noto Serif SC，编号/元信息走 Fraunces / JetBrains Mono。
- 竖版为**手机看图**媒介：正文字号显著大于网页版（1080px 图在手机上约缩到 0.36，故正文 30px+ 才够清晰）。

### 12.2 封面 `.cover2` · 单钩子大字报（改版定稿）

> **改版**：初版封面（刊名 146px 主角 + 3 条罗列）feed 点击率低——最大字给了品牌名、缩略图读不清 3 条小字。改版把封面赌在**当天头条 1 的钩子大字**上。基准原型 [prototype/xhs-封面-单钩子大字报.html](prototype/xhs-封面-单钩子大字报.html)（设计对话定稿、用户确认）；诊断/行为见 [design-briefs/小红书封面改版](../docs/design-briefs/小红书封面改版-单钩子大字报.md) + [spec §4](../docs/spec/小红书图组导出.md)。**完全复用 §1 token，零新增裸值/新色，红全来自 `--seal`。** 结构 flex 竖列三段：

- **① 品牌骨架条 `.cv2-head`（顶部 · 每天不变，padding 64 80 30，底 3px `--rule-2`）**：左 `.cv2-brand` = 26px `--seal` 方块印记 + 刊名「AI科技日报」（Noto Serif SC 900 **36px** `--ink-2`，**降级为角标、不再是主角**）；右 `.cv2-date` = `.d` 日期（mono 700 **40px** `--seal`）+ `.dow` 星期（Noto Serif SC 900 28px `--ink-3`）。品牌识别稳定、不抢大字。
- **② 大字钩子区 `.cv2-hook`（中部 · 主视觉 · `flex:1`+竖向居中，padding 0 80，占满方形安全区）**：
  - `.cv2-kicker`：mono 700 26px/.14em、**`--paper` 字 on `--seal` 实心 chip**、圆角 6（栏目 + 「今日头条」，如「AI 存储 · 今日头条」）。
  - `.cv2-big` **大字**：Noto Serif SC 900、line-height 1.14；**字号档位**——`.l` **116px**（≤16 字，2 行，存档备用）/ `.m` **94px**（17–24 字，3 行，**v1 默认**）；`em` = **数字/主体重音**（只换 `--seal` 色，**不改字重字号字体**）。
  - `.cv2-sub` **支撑句**：Noto Serif SC 500 **40px**/1.42 `--ink-3`（`em` 朱红 700 重音）；大字下补一句上下文事实，仍单主题。
- **③ 底部条 `.cv2-foot`（非安全区 · padding 30 80 64，顶 3px `--rule-2`）**：`.bar` 120×6 `--seal` 短横 + 一行 `.row`（mono 700 24px）——左 `.more`「+2 条头条 · N 速览 · 约 M 分钟」`--ink-3`，右 `.swipe`「左滑翻阅」`--seal` + 三连右向 chevron（**静态 SVG，非 emoji**）。**此条落 feed 方形安全区之外，缩略图裁掉无所谓。**
- **方形安全区**：feed 缩略图正方取中裁切 = 竖图中段（y 约 180–1260）。**大字钩子必须整句落在安全区内**；品牌条在其上、底部条在其下（可被裁）。
- 字号档（L/M）按大字实际字数**程序侧确定性选择**（非 AI）；大字/支撑句文案来源与反漂移见 [spec §4.1/§7](../docs/spec/小红书图组导出.md)。

### 12.3 头条页 `.headline`（padding 58 72 86 · flex 竖列）—— 方案 A

事实与 AI 判断的视觉分工（呼应 §9 事实/AI 契约）在竖版的落地：**事实＝裸通栏正文当主角，AI 分析＝朱红批注卡当从属**，一眼分主次、不互抢。

- 内容区 `.hl-body`：`flex:1` + `justify-content:center` —— **竖向居中**，短文案上下留白均分，不在底部堆空白。
- `.kicker`（栏目）：mono 700 26px/.16em `--seal` 大写。
- `h2` 标题：Noto Serif SC 900 **56px**/1.18。
- **事实块 `.fact`（主角）**：标签 `.label`「发生了什么」mono 700 26px `--ink-3`；正文 Noto Serif SC 500 **36px**/1.5 `--ink-2`。**无边框、无底色**（脱框才是主角）。
- **AI 分析 `.impact`（从属·批注卡）**：`--ai-bg` 底 + **左 8px `--seal` 竖条**；标签 mono 700 26px `--seal`，前置 `.chip`「AI」（`--paper` on `--seal`、圆角）作显式标注；正文 Noto Serif SC 500 **34px**/1.52 `--ink-3`（比事实小 2px，强化从属）。
- `.src` 来源：mono 26px `--muted`。
- **一条头条一张图，绝不拆图**（spec §2）。

### 12.4 速览页 `.briefs`（padding 60 72 86）

- `h2`「今日速览」Noto Serif SC 900 **66px**。
- `.brief-list`：上下各 3px `--ink` 粗线夹住整栏（简讯栏感）。
- `.brief-item`：`grid 64px 1fr` / gap 22 / padding 22 0 / 底 1px `--rule`（末条去线）。`.no` Fraunces 900 42px `--seal`；`h3` 标题 Noto Serif SC 700 **38px**/1.22；`p` 一句摘要 500 **31px**/1.4 `--ink-2`；`.src` mono 20px `--muted`。
- **去掉旧的底部「N 条快扫」footer**（只留页脚页码）。
- **高度驱动分页**（视觉约束，落地照此）：每页 ≤5 条，且**按实测高度分页**——放大字号后一页约容 **4 条长条 / 5 条短条**；单条总高超过页面可用高约 1/4 即拆入下一页。**宁可多一页留白，也不硬塞、不裁切**（对应 spec §2 动态分页）。

### 12.5 字数契约（设计推导值 · 待 [spec §7](../docs/spec/小红书图组导出.md) 回填 —— 产品对话定稿）

由上面定稿版面的可用空间推出，供产品对话回填 spec §7、作 AI 收敛输入依据。**权威在 spec §7**，此处仅记设计推导：

| 卡片·槽位 | 推导区间 `[min,max]`（字） | 依据 |
|---|---|---|
| 头条·标题 | 不收敛，理想 ≤36（2 行）、上限 ~52 | 56px |
| 头条·发生了什么 | ~[90, 155] | 36px 通栏、居中填充 |
| 头条·为什么重要·AI 分析 | ~[85, 145] | 34px 批注卡 |
| 速览·标题 | 不收敛，理想 ≤22（1 行）、上限 ~40 | 38px |
| 速览·一句摘要 | ~[22, 52] | 31px |

> 附加约束（供 spec）：速览**单条总高 ≤ 页面可用高 1/4**，否则拆页。真实字数须由 Codex 渲染真图核定，允许 ±3~5 字微调后冻进配置。
