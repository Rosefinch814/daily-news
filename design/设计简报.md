## 1. 这是什么

一份**科技板块的个性化电子报**（HTML 网页报刊），给作者自己看。每天手动生成一期，分**头条精读**和**速览**两层。完整产品方案见 [docs/spec/01-科技日报.md](../docs/spec/01-科技日报.md)。

## 2. 必须遵守的铁律（影响设计）

- **🔑 全程中文**：每条都中文可读，英文原文链接仅"备查"，不是必经。
- **📄 静态网页 · 无后端**：纯静态 HTML，没有服务器。→ 任何"实时交互"（点赞回传、实时调 AI）都做不到，得想替代方案。
- **🚫 忠于原文**：内容由 AI 生成但忠于事实；AI 判断（影响分析）要在视觉上**和事实区分开**。
- **📰 文风**：专业简报体（信息密度高、精炼专业，像高端 newsletter）。
- **🎨 风格参考**：用户朋友的 ai-weekly 是 neo-brutalist（黑框块状、无渐变）——可参考，不强制。

## 3. 一期的版面（产品已定的骨架）

```
报头（我的日报·科技 / 日期·期号）
📝 今日导读（AI 一段当天综述）
⭐ 头条精读 3-5 条（编号 头条1…）
⚡ 速览 10-15 条（编号 速览1…）
📡 关于本期（抓了几源/几条 → 留几条）
💬 反馈区（按编号点评 + 一句话）
← 上一期 · 📚 归档 · 下一期 →
```

**头条卡片字段**：编号 · 中文标题 · 摘要 · 〔展开〕精读 · 📊影响(AI,标注) · 来源·原文↗
**速览条目字段**：编号 · 中文标题 · 短摘要 · 〔展开〕精读(可选) · 来源·原文↗

## 4. 待设计对话定的"交互/功能"问题（重点）

1. **展开精读**：头条/速览的"精读"怎么展开——手风琴折叠？弹窗？锚点跳转？需不需要 JS（静态页可以带少量 JS）？
2. **速览的"按需精读"**：静态页没法实时调 AI。那速览的精读是**预生成好藏在页面里**（点开即显），还是**速览干脆不给精读**？这会反向影响内容生成。
3. **反馈区**：无后端下，反馈怎么呈现？倾向"一段说明 + 一个按编号写点评的文本框（供你复制出来喂给下次生成）"。具体形态待定。
4. **归档 / 上下期**：纯静态靠相对链接跳历史 HTML 文件？归档列表页怎么组织？
5. **设备**：只在电脑看，还是手机也要（响应式）？
6. **单页 vs 多页**：一期一个 HTML 文件？精读内容内联还是分离？

## 5. 待设计对话定的"视觉"问题

排版网格、配色、字体、头条/速览的视觉层级、"📊影响"如何视觉标注、整体风格基调（是否走 neo-brutalist）。

## 6. 产出去向

- **交互/页面行为** 的结论 → 产品对话并进 [spec](../docs/spec/01-科技日报.md)。
- **纯视觉设计**（配色/排版/风格/组件样式）→ 产品对话整理成 `design/` 下的设计文档。
- 最终都由 Codex 按文档落地成 `web/` 代码。
