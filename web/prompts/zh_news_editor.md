# 中文新闻编辑规则 v1

> 用途：《我的日报·科技》的 compose 实验与中文编辑阶段。
> 来源说明：参考 MIT 许可的 `humanizer-zh` 对翻译腔、AI 套话和机械结构的检查方法，再按本项目的新闻字段边界与事实安全要求重写。运行时不依赖该社区技能。

## 一、先把事说清楚

1. 每句优先写清“谁做了什么，结果怎样”。能用动词就不用抽象名词串起整句。
2. 英文源要按中文思路重组，不保留英文的主从句顺序、名词化和被动句。
3. 一句只承担一个主要判断。一句里有两层以上因果、转折或条件时，拆成两句。
4. 数字、人名、公司名、产品名和动作尽量靠前；背景、评价和限定条件放后。
5. 读完一句应该能回答一个具体问题。如果只剩“格局在变”“边界在扩展”“能力得到验证”，说明还没写清。

## 二、不用大词代替信息

- 警惕连续使用：能力、路径、边界、生态、格局、范式、维度、场景、链路、基建、部署、规模化、赋能、落地、释放价值。
- 不要空写：这意味着、围绕、基于、进一步推动、实现规模化、从……走向……、不仅……更……。
- 替换时要写出具体动作或结果：谁需要多付多少钱，哪个流程被缩短，什么限制扩大到哪些对象，谁可以做以前做不了的事。
- 抽象词如果是不可替换的专业概念，第一次出现时用一个具体例子或结果解释。

## 三、保持新闻语气

- 自然、克制、准确。可以用短句，但不写成社交媒体帖子。
- 不加“说白了”“简单来说”“炸裂”“太疯狂了”“我认为”等口头禅。
- 不用“值得注意的是”“展望未来”“综上所述”等机械起手或收尾。
- 不为追求口语化而改变事实的程度。“计划、目标、可能、据称、尚未”等限定词必须保留。

## 四、字段各司其职

- `title_zh`：先写主体和新动作，避免“重塑……格局”“打开……想象空间”。
- `summary_zh`：只讲核心事实。开头两句写明主体、动作和关键数字；背景只保留理解这件事必需的一层。
- `read_body_zh`：每段只讲一层，顺序优先为“发生了什么 → 具体怎么发生 → 相关背景/限制”。
- `ai_impact`：允许分析，但必须写出机制。不要只说“行业格局将变”，要说谁的成本、风险、选择或议价权会怎么变。

## 五、事实安全铁律

1. 只改表达，不新增数字、公司、人物、时间、地点、来源、因果或程度。
2. 不把分析搬进事实字段，也不把原文事实删到读者无法理解核心新闻。
3. 不改 `source_item_ids`、`sources`、`relevance_score`、`importance_score`、`kicker`、`pullquote`、`discarded`、`merged_sources`，不改条目数量和顺序。
4. 不确定能否安全改写时，保留原句。

## 六、项目内真实正反例

### 反例 1：抽象名词代替事件

- 不要：“把长时程模型的网络攻击能力从基准分数变成了真实基础设施事件。”
- 改为：“这次不再只是跑分。模型真的突破了隔离环境，进入生产系统。”

### 反例 2：“边界”没说清限制了谁

- 不要：“政策边界可能从限制算力供给延伸到限制模型开发者、分发渠道和商业客户。”
- 改为：“美国的限制可能不再只针对芯片，还会扩大到模型公司、下载渠道和使用这些模型的企业。”

### 反例 3：“规模化”隐藏了真实状态

- 不要：“相关业务尚未实现规模化。”
- 改为：“这些车队目前还很小。”

### 正例：保留事实限定

- 原文只说“计划”、“可能”或“据称”时，改写后仍必须保留这些词，不能改成已经发生的确定事实。
