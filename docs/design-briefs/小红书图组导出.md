# 设计简报 —— 小红书图组导出卡片

> 给**设计对话**（另一个 Claude 对话，frontend-design 技能）的输入。目标：定稿封面/头条/速览三种竖版卡片的视觉模板，**并反推出每个文本槽位的字数区间**。
> 权威约束见 [spec/小红书图组导出.md](../spec/小红书图组导出.md)；视觉基调见 [design/设计规范.md](../../design/设计规范.md)（V1a 报纸感）。设计对话是探索性的，**结论回产品对话成文**：纯视觉进 `design/`，行为/字数契约进 spec。

## 要解决的设计问题

1. **三种卡片模板**（竖版 1080×1440）：封面、头条页、速览页，风格统一成一套「报纸感」图组。
2. **★ 每个文本槽位的字数区间 `[min, max]`** ——这是本次设计的**特殊交付**：定稿版面后，量出每个槽位在保证不溢出/不难看的前提下能放多少字，产出 spec §7 的字数契约表。**这张表就是后端 AI 收敛阶段的输入依据**，务必给。
3. 报纸感与竖版新媒介的**取舍**：哪些沿用 design 规范 token、哪些需要 XHS 专属（成心新增、命名、落进 design 规范，不散写裸值）。

## 硬约束

- **封面**：不要大 masthead / 不要 Tourbillion News、Technology Daily；突出「今日科技日报」；**日期醒目**；「几条头条 / 几条速览 / 约几分钟」做成 **pill**（不是长句）；列今日 3 条头条标题；**简洁清晰、不能更乱**。
- **头条页**：一条头条一张图（绝不拆图）；分区清晰＝标题 / 发生了什么 / **为什么重要·AI 分析（显式标注、与事实视觉分离）** / 来源；字号比初版最大那版**小一号**（靠字数区间保证内容说完整，不靠狠截断）。
- **速览页**：每页 ≤5 条；每条＝标题 / 一句摘要 / 来源；字号比初版**大一号**；**去掉**底部「N 条快扫」footer。
- **通用**：正文读起来是完整句，不截断、不省略号（这要求版面能容下「收敛到区间」的文案）。

## 现状（可作为改造起点，别从零画）

Codex 已有一版可截图的实现：`web/src/daily_news/xhs_export.py` 里内联 CSS（约 395 行起），HTML+CSS → Playwright 截图。它的**结构骨架可参考**（封面 pill、头条 fact/impact 分区、速览列表），但：
- 文案是规则式截断（要废弃，换成字数区间）；
- CSS 自造了 paper/ink/seal 裸值、没引 design 规范 token（要和解）；
- 用户对它的最新评价是「有点乱了」——头条信息密度过高、事实块与 AI 分析块视觉抢、速览放大字号后像长列表、节奏乱。**这些正是要在设计层解决的。**

## 交付物

1. 三种卡片的视觉方案（版式、层级、字号阶、留白、报纸感元素、与 design 规范 token 的对应/新增）。
2. **字数契约表**（spec §7 的五个槽位各给 `[min, max]`）。
3. 头条页「事实块 vs AI 分析块」的视觉分工方案（让读者一眼分清、又不互相抢）。
