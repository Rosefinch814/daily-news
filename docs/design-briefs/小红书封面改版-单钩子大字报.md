# 设计简报 —— 小红书封面改版（单钩子大字报）

> 给**设计对话**（另一个 Claude 对话，frontend-design 技能）的输入。目标：把小红书**封面**从"报纸头版罗列"改成"单钩子大字报"，提升 feed 缩略图的点击率。
> **只改封面**——头条页 / 速览页（design §12.3 / §12.4）本轮不动。
> 权威约束见 [spec/小红书图组导出.md](../spec/小红书图组导出.md)；视觉基调见 [design/设计规范.md](../../design/设计规范.md)（V1a 报纸感）+ 现封面定稿 §12.2。设计对话是探索性的，**结论回产品对话成文**：纯视觉进 `design/§12.2`（改写），行为进 spec §4。

## 为什么改（问题诊断）

小红书曝光/点击率偏低。诊断：**封面是点击率杠杆点**（点击率低 → 算法停推 → 曝光低）。现封面（[原型](../../design/prototype/xhs-图组-科技日报.html) `.cover`、design §12.2）的硬伤：

1. **最大字号（146px）给了品牌名「AI科技日报」**——品牌名零钩子，最贵像素浪费。
2. **方形安全区里装的是"刊名+日期"**，不是"当天最劲爆的一句"，feed 里 0.5 秒扫过没有点击理由。
3. **3 条头条 40px 在缩略图里读不清**——三条并列既过载又都看不见。
4. **报纸感米白+墨黑在 feed 里不跳**，朱红面积太小。
5. **每天长一张脸**（结构固定、只变日期），易生"看过了"错觉。

一句话：现封面是"报纸头版"，不是"小红书封面"。

## 本轮方向（已拍板）

**单钩子大字报**：把**当天最有料的一条**，用**超大字**当封面主视觉、占满方形安全区；刊名 +日期**缩成角标**（品牌仍在，不占 C 位）；3 条罗列**删掉或压成极次要**。

> 弹药已有：T14 的「AI 钩子标题」本就在挑"当天最有料的角度"，可作封面大字的来源——但封面能容纳的字数/断行与 note 标题（≤20 字硬限）不同，二者关系见「开放问题」。

## 要解决的设计问题

1. **大字主视觉的排版系统**：一句钩子（长短不一）如何断行、如何**字号自适应**（钩子短→更大、长→自动缩档）稳定占满方形安全区，不会短了空、长了挤。给出字号档位规则，而非单一定值。
2. **钩子里的"数字/主体"要不要特殊放大**（如「38 亿」「SK 海力士」做视觉重音），怎么做既跳眼又不花。
3. **刊名 + 日期降级成角标**：放哪、多大、什么形态（页眉条？角标？），既保留品牌识别、又绝不抢大字。
4. **3 条头条的去留**：倾向**删**（feed 里读不清、给大字让位）；若保留只能是"缩略图看不清也无所谓"的次要暗示（如底部「+2 条 左滑」）。给出取舍建议。
5. **在报纸感基调里把对比度拉起来**：让封面在高饱和信息流里跳出来——朱红更大面积做视觉锤？底色/反白？在**不背叛 V1a 报纸感、不新增裸值**前提下探索。
6. **稳定骨架 vs 每日差异**：大字每天不同天然解决"长一张脸"；但仍需一套稳定的品牌识别骨架（角标位、朱红语言、字体），让人一眼认出是本刊。

## 硬约束

- **基调仍是 V1a 报纸感**，**完全复用 §1 token，不新增裸值/新色**（沿用现封面的和解结论）。
- **尺寸 1080×1440**；关键信息（大字钩子）必须落在**上部方形安全区**（feed 正方形取中裁切后仍完整可读）。
- **大字钩子是对外发布的事实性文案，反漂移铁律照旧适用**（spec §3.3）：忠实、不标题党、不新增/改写事实、不半句、不省略号。设计要能容下"完整一句钩子"，不靠狠截断。
- 只动封面；页脚「AI科技日报 · 日期」的品牌+日期语言可复用，别另造一套。
- 刊名文字仍是常量「AI科技日报」（`XHS_PUBLICATION_NAME`），设计不改字，只改它在封面的**权重与位置**。
