# 设计简报 —— 小红书封面 v2（数字/关键词做超大视觉主体）

> 本文是**已完成的设计探索输入**，记录 v2 从数字 hero 方向一路收敛的过程。最终落地结论以 [design §12.2](../../design/设计规范.md) 与 [spec §4](../spec/小红书图组导出.md) 为准；下文被后续否掉的 hero / 情绪前缀 / 关联签不是开发要求。
> **只改封面**——头条页/速览页（design §12.3/§12.4）不动。
> 权威约束见 [spec/小红书图组导出.md](../spec/小红书图组导出.md)（§3.3 反漂移、§4 封面、§7 字数契约）；视觉基调见 [design §12.2](../../design/设计规范.md)（V1a 报纸感）。基准原型 = [prototype/xhs-封面-单钩子大字报.html](../../design/prototype/xhs-封面-单钩子大字报.html)（T15，v2 沿用其设计基因，**但以独立模板实现、不覆盖 T15**）。

## 为什么迭代（问题 + 方法论）

T15 单钩子大字报上线后**点击率上升、点赞收藏变多**，方向成立。但用户实测两个新感受：

1. **冲击力还差**：大字 94px（M 档）相对 1080px 画布只占约 1/11 字高，缩略图里"能读清"但"不砸脸"。
2. **留白偏多**：`.cv2-hook{flex:1;justify-content:center}` 让内容垂直居中 → 大字"飘"在纸中段、上下各留一条空带（钩子越短越明显）。

**根因是同一个**：主视觉不够大 + 居中飘。**关键判断：留白本身不是错，"飘着的意外留白"才是错**——解法不是删空白，而是**让主视觉大到吃下空间、把剩余留白变成刻意边距**，一石二鸟。

**外部方法论（用户带回的小红书封面视频，已吸收）**：封面 = 信息流里的"判断器"，让人 0.5 秒判断"这是不是我要的"；顺序＝先信息层级 → 再点击理由 → 后审美；同赛道要一眼可辨。它戳中我们一个真短板：**"一眼知道这篇和我有关"**——陌生人刷到"6至10倍"不知道关自己啥事。故 v2 除了放大冲击力，还要补一层"关联/利益点"。

## 探索中途方向（后续已收敛，不作为落地要求）

- **主体 = 每日内容 hero**：从当日头条 1 的钩子里抽出**杀手元素（数字/短关键词）**做**超大朱红视觉主体**（如 `万亿日元`、`6至10倍`、`15倍`、`38亿`、`710亿美元`），句子降为支撑。**主体每天变**（保真实+新鲜），不换成固定痛点海报（那条已否）。
- **补一层固定"关联签"**：一句稳定的利益点，让陌生人秒懂"这号对我有啥用"。**固定小字、非新闻事实**，与内容主体分层。
- **报纸感 + 克制情绪锤（用户看完同赛道调研拍板）**：调研发现高点击封面都有"情绪/信息缺口"扳机(突发!/悬念/争议)，我们缺这个"勾"。落点＝在报纸感里加**克制的情绪前缀 + 关键词图形高亮**（详见设计问题 7/8），**仍逐字忠实、不标题党**；不走"emoji/疑问/手绘色块全上"的网感放开版。
- 仍守 V1a 报纸感：靠**超大衬线/展示体 type + 朱红**拉冲击，**不做色块/反白**（那是被否的方向）。

## 要解决的设计问题

1. **hero 版式**：超大数字/短词做主体，其余钩子（句子）+ 支撑句如何主次排布/环绕，占满方形安全区、把留白变刻意边距。给出 hero 与句子的层级关系（谁大谁小、上下还是包裹）。
2. **hero 字号自适应档**：hero 长短不一（`万亿` 2 字 vs `6至10倍` 5 字 vs `710亿美元` 5 字），给**字号档规则**（非单一定值），保证短的更冲、长的不溢出。
3. **★ 无数字/无短词兜底版式**：不是每天头条 1 都有干净 hero（如「苹果起诉 OpenAI」「谷歌重组 AI 团队」）。规则＝**有强数字/≤4 字关键词 → hero 化；否则退回"放大整句"版**（把当前 94px 整句放大、顶对齐、吃掉留白）。**须给两态版式**，程序按有无 hero 自动切——两态都要好看。
4. **留白重分配**：hero 吃掉中段空带，重心上移/顶对齐，剩余留白读作刻意边距而非"没填满"。
5. **feed 跳出度**：报纸感内把冲击力拉满——超大衬线 type + 朱红面积/权重，不靠色块/反白。
6. **★ 固定"关联签"（视频点 3）**：在**不抢 hero、不重新塞乱留白**前提下补一层"利益点/关联"信号。**建议**：把现有底部功能条（`+2 条头条 · N 速览 · 约 M 分钟`）升级成一句利益点（如「每天 3 条 · 帮你补齐 AI 重点」类）。探索**位置与形态**（底部升级 or 顶部 tagline）。可带轻悬念感（信息缺口），但**不标题党**。**文案先占位**——用户结合同赛道调研定稿。
7. **★ 克制情绪前缀（同赛道调研核心发现）**：调研发现高点击封面几乎都有一个"情绪/信息缺口"扳机（突发!/来了!/到底啥来头?/🤯）——我们平铺陈述事实、缺这个"勾"。**用户拍板走"报纸感 + 克制情绪锤"**：给 hero 可选一个**情绪/紧迫前缀**（如「突发」「重磅」「刚刚」），设计它的**视觉形态**（独占一行的小字？朱红标签 chip？多大、与 hero 的层级）——要抓眼但仍像报纸眉题、不像营销号弹窗。**触发规则/词表是产品行为，见硬约束 + 待回填 spec §4。**
8. **★ 关键词图形高亮（升级"只换色"）**：调研里关键词都用**图形标注**揪出来（黄手绘圈 / 红椭圆块 / 波浪线）。我们把 hero 关键词从 T15 的"仅换朱红色"**升级为朱红图形标注**（圈注 / 粗底线 / 衬色块选一，**报纸化处理、不要漫画手绘感**）。给出标注样式，仍只强调、不改字。
9. **引号视觉锚（可选）**：调研两张爆料/观点类用了大引号 `"` 做视觉锚 + 暗示"这是爆料"。评估是否给 hero 区加一个报纸感大引号装饰（纯视觉，不占语义）。

## 探索阶段硬约束（最终落地以权威文档为准）

- **复用 design §1 token，零新增裸值/新色，红全来自 `--seal`**（延续 T15 和解）。
- **反漂移铁律（[spec §3.3](../spec/小红书图组导出.md)）**：hero 的数字/词必须**逐字取自钩子**，不新增、不改写、不推断；句子/支撑句仍走 `xhs_condense` 收敛、收完整句、**无省略号**。关联签是固定营销文案、明确与新闻事实分区，不得混入或伪装成当日事实。
- **情绪前缀的笼子（关键 · 待 spec §4 正式化）**：① 只能从**固定有限词表**里选（如 `突发/重磅/刚刚/独家/首次`，写死常量、可配），**AI/程序不得自创词**；② **默认不加、宁缺毋滥**——只有当日头条 1 确属重大/突发才给，**拿不准就不加**（降级到无前缀 hero）；③ 前缀是"情绪标签"、**不得给常规新闻扣"突发"**冒充不实；④ 若由 AI 判定"够格+选词"，程序侧必须校验选词 ∈ 词表、否则丢弃——延续"AI 只从固定项里选、不改模板/不自创"的全项目安全边界。**这是"把真新闻说得更抓眼",不是标题党漂移。**
- **只改封面**：头条页/速览页（§12.3/§12.4、T12）不动。
- **与现有机制衔接、别另造**（供设计阶段理解落地可行性）：hero 来源拟**复用现有重音机制**——`xhs_condense` 的 `cover_hook` 输出已带重音标注（`emphasis_terms`），升级为"标出主 hero 片段 + 次要重音"，程序侧主 hero → 超大主体、无 hero → 退回放大整句版。**不新造 AI 任务。**
- **方形安全区**：hero + 句子 + 支撑句**整组**落竖图中段（y≈180–1260）；关联签若在底部则允许落非安全区（可被 feed 裁）。
- 字号档按内容**程序侧确定性选择**（非 AI）；`--no-ai-condense` 时走确定性兜底仍可复现。
