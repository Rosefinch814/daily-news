## 最大技术风险

**AI 能否按"关注清单 + 口味档案"，从一堆科技新闻里筛出"我想看的"，并随反馈变准。**

抓取（RSS/API）、渲染（HTML）都是成熟技术，风险低。真正不确定、决定产品成败的，是 **"精准 + 可调教"** 这两件事能否用我们选的方案做到。

## 待验证的关键假设

| # | 假设 | 怎么验 |
|---|---|---|
| A | 用 `claude -p`（Claude Code 无头模式）能稳定返回**结构化 JSON**（相关度/摘要/翻译/归类） | 写最小脚本喂 3–5 条新闻，看输出是否可解析、稳定 |
| B | AI 按**关注清单**筛选，结果"基本是我想看的" | 跑一期，人工逐条核对：该来的来了吗？噪音多吗？ |
| C | 把**口味档案**喂给 AI，能让筛选**朝反馈偏移** | 给一条"多看X少看Y"反馈 → 更新档案 → 重跑 → 对比前后 |
| D | 优先 RSS/API 的源，**抓取稳定**、正文可提取 | 逐源测试可达性与正文提取质量 |

## 验证方法（M1 跑通后）

1. `python .../main.py` 生成 `web/issues/<今天>.html`，浏览器打开。
2. **精准核对**：对照关注清单逐条问"这是我想看的吗 / 有没有该来没来的"。
3. **调教核对**：给一条"多看 X / 少看 Y"的反馈 → 重跑 → 看下一期是否朝偏好偏移。
4. **引擎核对**：确认 `claude -p` 输出可被脚本稳定解析（失败率、格式漂移）。

## 2026-06-23 Checkpoint MVP 实测

> 本轮先不调用 `claude -p`，由 Codex 代替 AI 阶段，目的是验证数据链路、文件结构、页面渲染和人工可读性。详细开发 runbook 见 `web/DEVELOPMENT.md`。

实测 run id：

```text
tech-2026-06-23-165511
```

实际流程：

```text
RSS 抓取
  -> 本地轻预筛
  -> Codex/AI 语义粗筛
  -> 全量正文补全
  -> Codex/AI 最终选题
  -> Codex/AI 中文日报结构
  -> 静态 HTML 渲染
  -> 人工打开确认
```

关键结果：

| 阶段 | 文件 | 本轮结果 |
|---|---|---|
| 抓取 | `01_raw_items.json` | 119 条有效新闻，0 失败 |
| 本地轻预筛 | `02_candidates.json` | 60 条 AI 输入池 |
| Codex/AI 粗筛 | `02_codex_shortlist.json` | 17 保留 + 10 备选 + 33 丢弃 |
| 正文补全 | `03_enriched_candidates.json` | 27 条候选全部抓到正文，0 失败 |
| 最终选题 | `04_selection.json` | 5 条头条，12 条速览 |
| 日报结构 | `05_issue.json` | 校验通过 |
| HTML 渲染 | `web/dist/` | `index.html`、`latest.html`、`issues/2026-06-23.html` |

本轮形成的技术结论：

- 纯规则粗筛不适合作为最终粗筛；规则应只做轻预筛，把候选压到约 60 条，再交给 AI 做语义判断。
- AI 粗筛需要处理英文理解、聚合快讯拆解、规则误判纠偏、同事件合并。
- 已经过 AI 粗筛后，正文补全应默认抓取全部 `keep + maybe` 候选，不应只抓前几条。
- 正文补全只应更新正文内容，不应再次用规则排序导致 AI 选中的候选被丢弃。
- v1 当前是纯静态 HTML 产物，公网访问不需要后端接口；数据库可暂缓，后续用于生成侧留档、反馈、口味档案和复盘。
- 本轮 HTML 已人工打开确认“日报还可以”，可继续进入自动化流程设计。

需要继续验证：

- 用 `claude -p` 替代 Codex 手工生成 `02_codex_shortlist.json`、`04_selection.json`、`05_issue.json` 后，结构化 JSON 是否稳定。
- 连续多天运行后，RSS 源稳定性、正文提取失败率、选题偏好是否稳定。
- 手机端视觉需要继续截图验收，当前只完成 HTML 结构断点检查。

## 技术选型（与假设对应）

- AI：`subprocess` 调 `claude -p`，要求 JSON 输出；先关键词粗筛降噪再交 AI 精排（控时延/成本）。
- 抓取：feedparser(RSS) + httpx + trafilatura/readability(正文)。
- 渲染：jinja2 静态 HTML。
- 配置：PyYAML。
