# Spec —— 01·科技日报

> 产品规格说明，研发应能照此实现而无需自行发明产品行为。会逐步更新。
> **基于 [MVP](../mvp/01-科技日报.md)（目标）派生，给出完整产品功能与方案。**
> 目标 / 验收 / 非目标见 [MVP](../mvp/01-科技日报.md)；做事步骤见 [里程碑](../milestones/01-科技日报.md)。

## 1. 问题 & 目标用户

**目标用户**：项目作者本人（单用户、自用）。

**问题**：用户每天被动刷信息流，① 会**漏掉**真正关心的内容；② 内容"别人喂什么看什么"，**不精准、不属于自己**。

**用户想要**：不是又一个新闻聚合器，而是 **一份属于自己的报纸**——板块由自己定义、每板块自己决定关注什么、并能通过反馈**不断调教、越来越懂自己**。核心难点是 **精准 + 越来越懂我**。

## 2. 核心概念：报纸 = 板块的集合

```
板块 = 名字 + 数据源 + 关注清单(种子) + 口味档案(会长大) + 反馈入口
```
- **关注清单**：第一天的你（手写种子，AI 筛选的初始尺子）。
- **口味档案**：被调教后的你（活的偏好记忆，每板块一份，互不污染）。
- 加新板块 = 多摆一个配置块，骨架不动。

本期只落"科技"一个板块。

## 3. 产品原则（铁律）

1. **🔑 全程中文可读**：绝不需要读英文原文；英文源一律由 AI 给中文，原文链接只作备查。
2. **🚫 忠于原文**：摘要/精读/翻译只能基于原文事实，不加料、不脑补；AI 的判断（如影响分析）必须**明确标注**。
3. **精准优先**：宁缺毋滥，要"我想看的"，不要大而全。
4. **分层**：每期 头条精读 + 速览。

## 4. 版面结构

```
报头（我的日报·科技 / 日期·期号）
  ↓
📝 今日导读（AI 一段当天综述，先给全局）
  ↓
⭐ 头条精读 3-5 条（编号 头条1…）
  ↓
⚡ 速览 10-15 条（编号 速览1…）
  ↓
📡 关于本期（抓了几源/几条 → 留几条，透明建信任）
  ↓
💬 反馈区（按编号点评 + 一句话"多看/少看"）
  ↓
← 上一期 · 📚 归档 · 下一期 →
```

## 5. 卡片字段

| 字段 | 速览 | 头条 |
|---|---|---|
| 编号 | ✅ | ✅ |
| 中文标题 | ✅ | ✅ |
| 中文摘要 | ✅ 短(2-3句) | ✅ |
| 〔展开〕中文精读 | ✅ 可点开（按需生成） | ✅ 默认更深 |
| 📊 影响（AI 分析，标注） | — | ✅ |
| 💡 为什么推给你 | — | ✅ |
| 来源 · 原文↗（备查） | ✅ | ✅ |

**头条分块结构**（事实与 AI 判断泾渭分明）：
```
[头条1] 中文标题
摘要：一两句把事说清。（事实）
精读：来龙去脉讲透——发生了什么、关键信息、背景。（事实，忠于原文）
📊 影响（AI 分析）：对竞争格局/你关注的公司可能意味着…（← 标清是 AI 判断）
💡 为什么给你：命中关注「英伟达 / AI芯片」+ 行业大事
来源 综合 The Verge、Reuters · 原文↗
```

## 6. 关注清单（你手写的种子）

```yaml
想看:
  公司: [英伟达, 苹果, 台积电, 特斯拉, 微软, 谷歌, 英特尔]
  主题: [AI芯片, 大模型进展, AI产品发布, 自动驾驶, 半导体]
  人物: [黄仁勋, 马斯克, 奥特曼, 库克]
不想看:                # 负向：命中即降权/丢弃，专滤噪音
  [小公司融资, 纯营销稿, 加密货币炒作, 谍照外观爆料]
```

## 7. 筛选逻辑（精准在这发生）

```
每条新闻 → AI 拿「关注清单 + 口味档案」打两个分：
   · 和你的相关度    · 事件重要度
        ↓
命中"不想看" / 太不相关 → 丢弃
相关                    → 速览
相关 且 重要            → 头条（3-5 名额，两分都高者胜出）
```
- **同源合并**：同一事件多源都报时，AI 在筛选时顺手捏成一条（标"综合 X、Y"）。本期不做独立去重系统。

## 8. 内容生成规范

| 任务 | 规范 |
|---|---|
| 文风 | **专业简报体**（信息密度高、精炼专业，像高端 newsletter） |
| 筛选打分 | 相关度 + 重要度双分；命中负向清单降权 |
| 中文摘要 | 每条 2-3 句，忠于原文 |
| 头条精读 | 一段把事讲透（事实）+ 标注的「📊影响」分析 |
| 为什么推给你 | 一句点明命中的关注点 |
| 翻译 | 英文源融进中文摘要/精读，原文仅备查 |
| 今日导读 | 一段综述当天最值得注意的，先给全局 |
| 红线 | 全程忠于原文，AI 判断必须标注，不编造 |

## 9. 调教闭环（灵魂）

- **反馈形式**：按**编号**的细点评（`头条2 👍 / 速览5 👎`）+ 一句话方向（`多看芯片，少看外观`）。
- **回传方式（本期）**：**方案 A 纯文本**——你把反馈写给下次生成（编辑文件/喂给脚本）；**网页不放点击按钮**（无后端，点了没人接）。以后再升级本地小服务让 👍👎 可点。
- **口味档案**：一份活文档（每板块一份），AI 把反馈消化成**偏好规则**（如"对英伟达兴趣最高""少看融资/营销稿"）。
- **如何生效**：下次筛选时 AI **同读「关注清单 + 口味档案」**打分 → 越用越准。
- **不训练/微调模型**，靠"反馈 → 口味档案 → 越来越准"的记忆闭环。

## 10. 用户流程（数据流）

```
配置(源 + 关注清单)
 → [抓取] 各源新闻 + 正文提取
 → [筛选] AI 读「关注清单 + 口味档案」打分(相关度+重要度) → 分层(丢弃/速览/头条)，同源合并
 → [加工] 全中文：摘要 / 头条精读 / 标注的影响分析 / 翻译 / 今日导读
 → [渲染] 生成当天 HTML 电子报（导读、头条、速览、源面板、反馈区、归档导航）
 → [阅读] 浏览器打开，全程中文
 → [反馈] 按编号点评 + 一句话"多看/少看"（方案A 纯文本回传）
 → [调教] AI 消化反馈 → 更新口味档案 → 下一期更准 ⟲
```

## 11. 数据输入与输出

**输入**：数据源清单（RSS/API，按板块）、关注清单（公司/主题/人物/不想看）、口味档案、用户反馈（编号点评 + 一句话）。
**输出**：当天 HTML 电子报、历史归档、更新后的口味档案。

## 12. 开发方案（高层架构）

- **语言**：Python。代码/运行时内容放 `web/`。
- **AI 引擎**：Claude Code 无头模式（`claude -p`），封装成**单一可替换函数**，返回结构化 JSON。
- **抓取**：优先 RSS/API，不爬付费墙。
- **渲染**：静态 HTML（Jinja2），无服务器。
- **运行**：手动跑命令生成。
- 组件（建议）：fetch（抓取+正文）/ ai_engine（筛选+生成）/ render（渲染）/ feedback（消化反馈）/ main（编排）。
- ⚠️ **具体技术栈细节（库选型、目录布局）待最终确认**（用户要求最后再定）。

## 13. 科技板块配置（种子）

**重心**：全景科技——AI/大模型、芯片/硬件、大公司动态/产业格局、产品/消费科技。

**数据源**（优先 RSS/API、不碰付费墙）：
- 英文：The Verge、Ars Technica、Reuters Technology、TechCrunch、Hacker News（科技综合）
- 中文：36氪、爱范儿

## 14. 开放问题

- 每期具体条数（头条 3-5 / 速览 10-15 区间内微调）。
- 口味档案的具体结构与"消化反馈"的更新策略——实现时定。
- 速览"展开精读"是按需生成还是预生成——实现时按成本定。
- 技术栈细节（库选型、目录）——用户要求最后再定。
