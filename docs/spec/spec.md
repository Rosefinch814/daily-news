# Spec —— 我的日报

> 本文是产品规格说明，研发应能照此实现而无需自行发明产品行为。会逐步更新。

## 1. 问题 & 目标用户

**目标用户**：项目作者本人（单用户、自用）。

**问题**：用户每天被动刷信息流获取新闻，存在两个痛点：
1. 会**漏掉**真正关心的内容（科技产业动态、影响科技股的大事等）。
2. 信息流是"别人喂什么看什么"，**不精准**、不属于自己。

**用户真正想要的**：不是又一个新闻聚合器，而是 **一份属于自己的报纸**——
- 由自己定义的若干**板块**组成；
- 每个板块里自己决定关注什么；
- 报纸能通过反馈**不断被调教、越来越懂自己**。

核心难点不是"抓新闻"，而是 **精准 + 越来越懂我**。

## 2. 核心概念：报纸 = 板块的集合

每个板块是一套**四件套**：

```
板块 = 名字 + 数据源 + 关注清单(种子) + 口味档案(会长大) + 反馈入口
```

- **关注清单**：第一天的你（种子偏好，AI 筛选的初始尺子）。
- **口味档案**：被调教一段时间后的你（活的偏好记忆，每板块一份，互不污染）。
- 加新板块 = 多摆一个配置块，骨架不动。
- "精准"靠每个板块**自己那把尺子**。

## 3. MVP 范围

**只做"科技"单板块**，手动生成。详见 [../mvp/mvp.md](../mvp/mvp.md)。

证明两件事：
1. **筛得准**（核心价值）：能从一堆科技新闻里挑出"我想看的"。
2. **能调教、越来越懂我**（最大技术风险）：给反馈后下一期朝偏好偏移。

## 4. 用户流程

```
配置(板块的源 + 关注清单)
        │
        ▼
[抓取] 拉取该板块各源的新闻 + 正文提取
        │
        ▼
[筛选] AI 读「关注清单 + 口味档案」→ 打相关度分 → 留高相关项
        │
        ▼
[加工] AI 摘要 + 英文源翻译 + 归类
        │
        ▼
[渲染] 生成当天 HTML 电子报（含编辑导读、"关于本期"源面板）
        │
        ▼
[阅读] 用户浏览器打开
        │
        ▼
[反馈] 👍/👎 或一句话「多看X少看Y」
        │
        ▼
[调教] 消化反馈 → 更新该板块口味档案 → 下一期更准
```

## 5. 数据输入与输出

**输入**
- 数据源清单（RSS/API 地址，按板块归属）。
- 关注清单（公司 / 主题 等，种子偏好）。
- 口味档案（活的偏好记忆，随反馈生长）。
- 用户反馈（👍/👎 / 自然语言）。

**输出**
- 当天 HTML 电子报（分板块、含摘要/翻译/相关度、编辑导读、源面板、反馈入口）。
- 历史归档（按日期）。
- 更新后的口味档案。

## 6. 技术约束

- **语言**：Python。
- **AI 引擎**：复用 **Claude Code 无头模式**（`claude -p`），不另接 API key。封装成**唯一可替换函数**（`ai_engine`），以后可切 API。
- **抓取**：优先 **RSS/API，能不爬就不爬**，付费墙直接跳过。
- **AI 任务**：筛选 / 摘要 / 翻译 / 归类，要求返回**结构化 JSON**；可先关键词粗筛降噪，再交 AI 精排。
- **渲染**：静态 HTML（Jinja2），无需服务器。
- **运行**：MVP 手动跑一个命令生成；定时自动化放后期。
- **调教机制**：**不训练/微调模型**，用"反馈 → 口味档案 → 越来越准"的记忆闭环。

## 7. 验收标准

1. **精准**：对照关注清单逐条问"这是我想看的吗 / 有没有该来没来的"，主观判定"基本是我想看的"。
2. **调教**：给一条"多看 X / 少看 Y"的反馈 → 重跑 → 下一期筛选可见地朝偏好偏移。

## 8. 非目标（MVP 阶段明确不做）

- 财经/宏观/AI 学习等其他板块（科技跑通后再复制）。
- 定时调度、邮件/推送。
- 精美视觉打磨。
- 跨源去重、源失败容错等边界处理。
- 多板块管理 UI。
- 多用户、登录、云端。

## 9. 开放问题

- 关注清单种子的精确内容（公司/主题）以哪份为准——见 [科技板块配置](#) / 计划文件，第一期后据反馈调。
- 每期量级（精华 5 条 / 速览 10–15 条 / 分层）——尚未最终敲定，先实现可配置。
- 口味档案的具体结构与"消化反馈"的更新策略——阶段 1 设计时定。
- 反馈入口形态（静态 HTML 如何收集反馈：先用最简方式，如手写一句话喂给脚本）。
