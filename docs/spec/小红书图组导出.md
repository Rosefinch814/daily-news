# Spec —— 小红书图组导出（export-xhs）

> **本功能的产品行为唯一权威。** 把某一期已发布的科技日报 JSON，导出成一组适合小红书发布的竖版图片 + 一段 caption。
> 定位：**认真做成一个内容渠道**（用户拍板），值得投入独立设计阶段 + 一个受控 AI 阶段。
> 归属里程碑：M4「分享」项前移（见 [milestones](../milestones/01-科技日报.md)）。上游日报行为仍以 [01-科技日报 spec](01-科技日报.md) 为准，本文只管「日报 JSON → 小红书图组」这一段下游导出。

## 0. 一句话

`daily-news export-xhs --date YYYY-MM-DD` 读已发布的 `web/dist/data/issues/<date>.json`，产出封面 + 头条 + 速览若干张竖图（1080×1440）和一段 `caption.txt`，**不改主 pipeline、不改网页日报、不影响 Cloudflare 发布版本**。

## 1. 边界与非目标（先划清）

- **纯下游、只读**：输入是**已发布的** issue JSON（`dist/data/issues/`），不回改任何日报数据、不进 `run-pipeline` 阶段链、不触发部署。
- **独立命令**：`export-xhs`，与生成/消化链路解耦，随手对某一天跑。
- **旧项目无关**：`web/marketing/xiaohongshu/` 是历史项目，**不复用、不参考、不改动**。本功能代码在 `web/src/daily_news/xhs_export.py` 一线。
- **本期不做**：自动发小红书（发布交互归后续，导出的图 + caption 先手动发）、AI 生成封面配图/插画、多板块（先只科技）、把导出接进定时。

## 2. 图组结构（已定倾向）

图片数量**不固定**，按内容多少动态。顺序恒定 `封面 → 头条1 → 头条2 → 头条3 → 速览`：

1. **封面 ×1**：由 `--cover-template` 选择 `classic` / `single-hook` / `v2`，三个模板并存且产物目录隔离，定义见 §4。`single-hook` 与 `v2` 的主视觉都锁定**当天头条 1**，不在封面罗列 3 条头条；总览交由 [caption 正文](#9-小红书文案发布用--标题--正文)。`classic` 保留旧版展示，不因新模板变更。
2. **头条 ×最多 3**（一条头条 = 一张图，绝不拆成多图）。`single-hook` / `v2` 封面与头条 1 同主题、前后呼应。
3. **速览动态分页**：每页**最多 5 条**，按标题+摘要长度动态分页；页数随内容（示例某天可能 3 页）。

> 页数动态是刻意的：宁可多一页留白舒服，也不硬塞。头条恒定「一条一图」。
> `single-hook` / `v2` **不设总览卡、不罗列 3 条**：把封面注意力全押在头条 1 上（feed 缩略图只够承载一个强钩子），是为点击率的刻意取舍，见 [design-briefs/小红书封面改版](../design-briefs/小红书封面改版-单钩子大字报.md)。

## 3. 文案怎么来（核心机制：版面定区间 + AI 轻度收敛）

这是本功能与 Codex 初版最大的差别，也是产品命门。**废弃**初版那套 `headline_budgets`/`brief_budget`/`fit_complete_text` 规则式动态截断。

### 3.1 两步契约

1. **版面 → 字数区间（设计阶段定，回填到 §7 契约表）**：每种卡片的每个文本槽位（头条标题 / 头条·发生了什么 / 头条·AI 分析 / 速览标题 / 速览摘要），由定稿版面推出一个**目标字数区间 `[min, max]`**——这是确定的、写死在配置里的契约，不再每条现算。
2. **AI 轻度收敛到区间**：新增一个受控 AI 任务，输入=日报原文该槽位文本 + 该槽位的 `[min, max]`，输出=**忠实收敛**到区间内的文本。

### 3.2 AI 收敛任务（`xhs_condense`）——契约

- **入口**：走现有可替换 `ai_engine` 单一入口（`CLAUDE_COMMAND` 间接层），**绝不把某种调用方式焊死进导出逻辑**；provider 可配（同 pipeline 其它 AI 阶段）。
- **输入 payload（每槽位）**：`{slot_type, title(上下文), original_text, target_min, target_max}`；`slot_type ∈ {cover_hook, cover_sub, headline_summary, headline_impact, brief_summary}`（头条/速览标题一般不收敛，超长另议，见 §7）。`cover_hook`/`cover_sub` 取自头条 1 的标题/摘要，见 §4.1；`cover_hook` 可在输出里附带**重音片段标注**（数字/主体），由不同模板决定是包裹 `<em>` 还是 `.mark`，无标注时程序侧正则兜底。
- **输出**：结构化 JSON `{text}`。允许一次批量收敛整期所有槽位、返回映射（省 AI 调用），但语义按单槽位定义。
- **就近性**：若原文已落在 `[min, max]` 内，**原样返回**，不无谓改写。

### 3.3 反漂移铁律（不可谈判）

与全项目「事实不漂移」一脉相承：

- AI **只做收敛（压缩到区间）**，不得新增、改写、推断任何事实、数字、主体、时间、来源。
- 收敛必须**收到完整句**，读起来不能像没说完；**绝不输出 `…` / `...`**。
- **来源署名、关键数字、主体公司/人物不可动。**
- **AI 分析块必须显式标注**（如「为什么重要 · AI 分析」），与事实摘要视觉分离，不得与事实混为一谈。
- 冲突时的取舍顺序：**事实忠实 ＞ 版面齐整**。某条实在收敛后仍不完整/放不下 → **降级而非漂移**：速览该条移到下一页或本期不收进图组；头条本就最多 3 条、有取舍空间。宁可少放一条、留白，也不编内容凑版面。

## 4. 封面模板与版本隔离（行为定稿 · 视觉见 [design 规范 §12.2](../../design/设计规范.md)）

`export-xhs` 的封面模板通过 `--cover-template` 显式选择，**新版只新增，不覆盖旧版**：

| 参数 | 定位 | 默认输出目录 | 文案与视觉边界 |
|---|---|---|---|
| `classic` | 初版报纸封面 | `web/runs/xhs/<date>/` | **冻结现有行为**；仍是命令默认值 |
| `single-hook` | T15 单钩子大字报 | `web/runs/xhs/<date>-single-hook/` | **冻结 T15 视觉和文案逻辑**；继续只用 `xhs_condense` 的 `cover_hook` / `cover_sub` |
| `v2` | 标题即主视觉 | `web/runs/xhs/<date>-v2/` | T16 独立视觉；T17 的 `xhs_magnetize` **只对此模板生效** |

- 不传 `--cover-template` 时仍选 `classic`；显式 `--output-dir` 时仍优先使用用户指定目录。
- 三种模板只分叉封面；头条页、速览页、caption 与图组顺序保持同一套行为。
- **兼容性铁律**：T16/T17 不得改动 `classic` / `single-hook` 的 DOM、CSS、字号档、文案来源、AI 调用、校验或默认目录。

### 4.1 v2 版面与文案来源

> **设计沿革**：初版封面的 feed 点击率低 → T15 改为单钩子大字报后点击/收藏见涨 → v2 在不替换 T15 的前提下，作为独立模板继续验证「标题即主视觉」。视觉权威在 [design §12.2](../../design/设计规范.md) + 原型 `design/prototype/xhs-封面-v2-数字主体.html`。

**v2 版面三层（自上而下）**：

1. **品牌骨架条（顶部 · 每天不变）**：左＝刊名角标「AI科技日报」（降级为 36px 小字 + 朱红方块印记，**不再是主角**）；右＝日期角标（`2026.07.07` mono 朱红 + 星期）。一条 3px 报头分隔线收口。**目的：稳定的品牌识别，不抢大字。**
2. **大标题区（中部 · 主视觉 · 占满方形安全区）**——封面的命门（**v2「标题即主视觉」**，视觉见 design §12.2）：
   - `eyebrow`（品类眉题）：栏目 + 「今日头条」**朱红平文红字**（v2 从实心红章降为平文，不抢标题；如「AI 公司 · 今日头条」）。**栏目名取自头条 1 的既有栏目字段，无则只留「今日头条」，不 AI 现编。**
   - **大标题**＝**头条 1 标题磁化·克制版**（见 §4.2），Noto Serif SC 900 **100px**、**标题本身即主体、数字不再单独放大**；过长则程序侧确定性下调字号（见 §7）。
   - **关键词高亮**：把大标题里的关键词做**朱红粗底线**（`.mark`，v2 升级 T15 的"仅换色"）——**一条即可、别滥标**，只强调不改字；来源为磁化/收敛顺带标注 + 程序侧正则兜底。
   - **支撑句**＝头条 1 摘要收敛（见 §4.1），40px、`--ink-3`；**放足两三行**上下文、靠内容长满版心，多给点击理由。
3. **底部功能条（非安全区 · 缩略图裁掉无所谓）**：一行——`+2 条头条 · N 速览 · 约 M 分钟` + 「左滑翻阅」+ 三连 chevron（**静态 SVG**）。**v2 去掉 T15 的红小节棒**。

- **v2 大字** = 头条 1 标题；T16 阶段先用 §7 的 `cover_hook` 收敛，T17 完成后默认用 §4.2 的磁化·克制版，磁化不可用/越界则回退 `cover_hook`。**v2 支撑句** = 头条 1 摘要的 `cover_sub` 收敛。§3.3 反漂移铁律始终照旧。
- **关键词朱红粗底线高亮（v2）**：由磁化/收敛任务在输出里**顺带标出**要强调的**一个**关键片段（数字/主体公司/人物），程序侧据此包裹 `.mark`（朱红粗底线，design §12.2）；**兜底 = 无 AI 或未标注时，程序侧正则识别数字串包裹**。**只强调、不改字、一条别滥标**。（T15 为"仅换 `--seal` 色"，v2 升级为粗底线。）
- 与 **caption 标题（§9.1）的关系**：caption 标题是 ≤20 字的 AI 钩子、可挑任意角度；**封面大字锁定头条 1**（头条 1 即当天选题最重要一条）。二者可不同措辞、允许分叉，都受反漂移约束。

### 4.2 v2 封面标题磁化（克制默认 + 冲版备选）

**目的**：封面大字是 feed 第一眼——在反漂移铁律内，把头条 1 标题从"新闻通稿体"改写得**更想让人点**（同赛道高点击封面靠"情绪/信息缺口"扳机，我们缺这个）。**这是非视觉的点击率杠杆**，与 §4 的视觉改版并行。

**来源升级（仅 v2）**：v2 封面大字来源从 §4.1 的「头条 1 标题只压不改收敛」升级为「头条 1 标题**磁化**」，走**新的可替换 `ai_engine` 任务 `xhs_magnetize`**（同 `xhs_condense`/§9.1 走可替换入口，provider 可配）。`classic` / `single-hook` 不调用该任务。

**事实层 / 表述层的线（§3.3 铁律细化，不可谈判）**：
- **事实层（一个字不许动）**：数字 / 金额 / 百分比、主体公司与人物、时间、来源、**程度**（"计划 / 目标"不得说成"已实现"，"逼近"不得说成"超过"）。
- **表述层（可优化）**：① 冲击数 / 反常点**前置**（别埋句尾）；② 去通稿腔、**说人话**；③ 突出**真实反差 / 痛点**（不编）；④ 留**信息缺口**（好奇但不骗）。

**两档输出（一次产两条）**：
- **克制版 = 默认渲染上封面**：报纸感忠实，不用情绪词、比喻克制。
- **冲版 = 存进产物、不自动发**：更冲，允许有限情绪 / 口语 / 比喻，仍守事实层。供用户**手动 A/B 测试**，测出好版本再回填口味。
- 原标题、克制版、冲版一起写入 `web/runs/xhs/<date>-v2/` 的变体产物；其他模板不生成该文件。

**护栏（程序侧硬校验，AI 之外再验一遍，杜绝"只靠 AI 自觉"）**：
- **数字校验**：磁化文本里出现的数字 / 金额 / 百分比，必须都能在原标题 + 摘要里找到（**禁新增、禁改数字**）。
- **主体校验**：主体公司 / 人物同理，不得替换或杜撰。
- **克制版禁用情绪词表**（可配常量，如 `震惊 / 太火了 / 暴涨 / 崩盘 / 吊打 / 秒杀`）——命中即判违规。
- **长度**：落 §7 封面大字区间。
- **任一不过 → 回退 §4.1 的 `cover_hook` 收敛兜底**（原标题只压不改），**永不发违规文案**，记一行日志。
- **开关**：`--no-ai-condense`（或等价）关掉时，v2 大字 = 原标题收敛，确定性可复现；不影响两个旧模板。

**few-shot**：prompt 模板内置**用户校准的正例 / 反例**（模板写死，AI 只填内容不改模板——延续全项目安全边界）。样板见工单 T17，锚点例：
- ✅ 正例：`智谱ARR半年增长15倍达10亿美元` → 克制「智谱ARR半年增长15倍，达10亿美元」/ 冲「半年翻15倍，智谱ARR冲到10亿美元」。
- ❌ 反例：「智谱**估值**暴涨15倍」（ARR→估值，改事实）；「**震惊！**智谱ARR暴涨」（震惊体 + 程度夸大）。

## 5. 头条页 / 速览页要求（设计定稿）

**头条页**保留且分区清晰：标题 / **发生了什么**（事实摘要）/ **为什么重要 · AI 分析**（显式标注、与事实分离）/ 来源。字号比初版「最大那版」小一号，靠 §3 的区间+收敛把内容说完整，不靠狠截断。

**速览页**：每条保留 标题 / 一句摘要 / 来源；字号比初版大一号；去掉「5 条快扫」之类底部 footer；每页 ≤5 条。

**两页共同**：不截断、不省略号、读起来是完整句（§3.3 铁律）。

## 6. 与设计系统的关系

- 竖版 1080×1440 卡片是**新媒介**，允许与网页日报有意分叉，但**基调仍是 [design/设计规范.md](../../design/设计规范.md) 的 V1a 报纸感**。
- 初版 CSS 自造了 paper/ink/seal 一套裸值——设计阶段须**和解**：能对齐 design 规范 token 的对齐，确需 XHS 专属的，**成心新增、落进 design 规范并命名**，不散写裸值。
- 卡片视觉模板（封面/头条/速览三种）定稿后：**纯视觉进 `design/`，导出行为/字数契约进本 spec**。

## 7. 字数契约表（设计推导值已回填 · 源 [design §12.5](../../design/设计规范.md)）

由定稿版面可用空间推出，作 `xhs_condense` 的输入契约与实现依据。**真实字数须由 Codex 渲染真图核定，允许 ±3~5 字微调后冻进配置**（工单 T12 落地为可配置常量/config，不散写魔法数）。

| 卡片 | 槽位 | 目标区间 `[min, max]`（字） | 备注 |
|---|---|---|---|
| **single-hook 封面** | **大字钩子** `cover_hook` | **[12, 24]** | T15 行为冻结；头条 1 标题只收敛不磁化；≤16 字→L 116px，17–24 字→M 94px；数字/主体朱红换色重音 |
| **single-hook 封面** | **支撑句** `cover_sub` | **[28, 46]** | T15 行为冻结；头条 1 摘要收敛；40px、一到两行 |
| **v2 封面** | **大标题** `cover_hook` | **[12, 24]** | 取自头条 1 标题（T17 后默认走 §4.2 磁化·克制版；磁化不可用则收敛兜底）；默认 100px，过长则程序侧确定性下调（100→90→82）；关键词朱红粗底线 |
| **v2 封面** | **支撑句** `cover_sub` | **[28, 46]** | 取自头条 1 摘要收敛；40px、**版式允许两三行** |
| 头条 | 标题 | 不收敛；理想 ≤36（2 行）、上限 ~52 | 56px；超长保留原题、版面自适应，不截 |
| 头条 | 发生了什么（事实摘要） | **[90, 155]** | 36px 通栏、竖向居中填充；AI 收敛 |
| 头条 | 为什么重要·AI 分析 | **[85, 145]** | 34px 批注卡；AI 收敛，显式标注 |
| 速览 | 标题 | 不收敛；理想 ≤22（1 行）、上限 ~40 | 38px |
| 速览 | 一句摘要 | **[22, 52]** | 31px；AI 收敛 |

> 附加约束：速览**单条总高 ≤ 页面可用高约 1/4**，否则拆入下一页（§2 高度驱动分页）。区间是软目标——落不进就按 §3.3 降级，不为凑字数漂移。
> **封面档位按模板分开**：`single-hook` 继续使用 T15 的 L/M 档；`v2` 默认 100px，仅在溢出时程序侧确定性下调。两套选档/降档逻辑不得相互覆盖。

## 8. 输入输出与技术落点

- **输入**：`web/dist/data/issues/<date>.json`（`Issue` 模型：`headlines[].{title_zh, summary_zh, ai_impact, sources}`、`briefs[].{title_zh, summary_zh, sources}`）。
- **输出**（`runs/` 已在 gitignore，属运行时产物）：`classic` → `web/runs/xhs/<date>/`，`single-hook` → `web/runs/xhs/<date>-single-hook/`，`v2` → `web/runs/xhs/<date>-v2/`；每个目录含 `01.png…`、`caption.txt`、中间 `cards.html`，v2 可另含 `cover_title_variants` 产物。
- **渲染机制**：HTML+CSS 卡片 → Playwright 无头 chromium 逐卡截图。Playwright 为本功能新增运行时依赖（`playwright install chromium`），限于导出侧，不进阅读链路。
- **发布文案（标题 + 正文）**：与图同批产出（整合进 `caption.txt` 或独立文件），内容规则见 §9。

## 9. 小红书文案（发布用 · 标题 + 正文）

`export-xhs` 除图外，同时产出发布用的 **标题** + **正文**（整合进 `caption.txt`：标题首行、空行、正文；或独立文件，实现定）。

**平台硬约束**：标题 **≤20 字**、正文 **≤1000 字**；超限必须收到完整句/换更短，**绝不硬发超限**。

### 9.1 标题 = AI 钩子（可回退）
- 挑当天最有料的角度，写一句 **≤20 字** 标题，**忠实、不标题党、不新增事实**（呼应 §3.3）。走可替换 `ai_engine` 入口。
- AI 不可用 / 超 20 字 / 明显跑偏 → **回退固定格式**「AI科技日报 · M月D日」。

### 9.2 正文 = 精简拼装（零 AI、零漂移）
**刻意短、不堆字**（全文已在图里，正文只作发现与钩子）。结构：
1. 话题标签行（几个固定标签，可配，如 `#AI日报 #人工智能 #AIGC #科技资讯`）。
2. 今日头条 **3 条标题**（编号，**逐字复制**自 issue，不改写）。
3. 一句品牌 slogan（固定，走刊名同源常量）。

- **速览不逐条列**（都在图里），至多一句带过。**目标 ~150–250 字，远低于 1000 上限**。
- 全部标题逐字复制、**不喂 AI**，杜绝漂移（§3.3）；hashtag / slogan / 刊名走配置常量，一处改。

## 10. 验收标准

1. **忠实**：对随机一期导出，逐张核对头条/速览正文，**信息与日报原文一致、无新增事实、无半句截断、无省略号**；AI 分析块标注清晰、与事实分离。
2. **版面**：三种封面模板均可导出；`classic` / `single-hook` 与 T16/T17 前完全一致；`v2` 满足 §4.1（平文红字眉题、100px 标题主视觉、朱红粗底线、支撑句两三行、页脚无红棒、溢出确定性降档）；头条一条一图、速览每页 ≤5。
3. **字数契约**：各槽位落在 §7 区间内；原文已在区间内则未被无谓改写。
4. **发布文案**：标题 ≤20 字且忠实（AI 挂了回退固定格式）；正文精简（hashtag + 头条 3 + slogan）、≤1000 字、目标 ~150–250 字、标题逐字复制无漂移。
5. **边界**：跑 `export-xhs` 不改动任何 `dist/`/pipeline 数据、不触发部署；输出只在 `runs/xhs/`；三种模板的默认目录不互相覆盖；`web/marketing/` 未被动。
6. **可替换**：AI 收敛 / 标题钩子 / v2 磁化均经 `ai_engine` 入口，切 provider 不改导出业务逻辑；`xhs_magnetize` 在 `classic` / `single-hook` 路径上调用次数为 0。
7. `pytest -q` 全过。

## 11. 开放问题

- 头条/速览标题超长（超出区间）时：收敛还是保留原题换行？倾向保留原题、版面自适应，具体由设计阶段定。
- `xhs_condense` 批量 vs 逐槽位调用的性价比与稳定性——实现时定，契约不变。
- 固定 hashtag / slogan 的具体文字由用户定稿后冻进配置常量（占位见 §9）。
