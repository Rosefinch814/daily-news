## 目标

用**最小、确定性、零 AI、零新依赖**的办法解决"今天的新闻和昨天重复"——把最近 N 天已发期出现过的条目，在本地预筛时直接过滤掉。权威行为见 [spec §7.5](../spec/01-科技日报.md)。

> 这是 M3"重复新闻不刷屏"的**先行最小版**：只杀"同一条原文还赖在源里"这种**绝大多数**的重复（同 URL）。相似度/事件指纹/跟进降权等更重的手段**本工单不做**，留到确认还有重复痛点时再议。不依赖 M2 的 T06/T07，可独立开发。

## 范围

1. 在**本地预筛阶段**加一道跨期过滤：拼出"最近 N 天已发期的 URL 集合"，候选 URL 命中就丢。
2.（可选第二档）在 URL 之上再加**标题归一化 hash** 去重，治"同一篇被不同 URL 重发/转载"。
3. 窗口 N 可配（默认 7 天）。**v1 不引入 AI、不新增 AI task_type、不加三方依赖、不引入相似度/指纹/penalty 分级。**

## 涉及文件

- `web/src/daily_news/scoring.py`：现有跨条目去重用的 `seen_urls`（只在单次运行内生效）——把它**用历史 URL 预先灌满**，让本地预筛顺手把"近 N 天已发过的 URL"一并滤掉。优先复用这条已有路径，不另起炉灶。
- `web/src/daily_news/storage/local.py`（建议）：加载近 N 期已发 issue（复用现有 `load_issue` / `web/runs/issues/` 目录），抽出其头条+速览的 `url`（第二档再抽归一化标题 hash），拼成历史集合。
- `web/config/`（建议）：窗口 N、是否启用标题 hash 档，放进现有 pipeline/config，不硬编码。
- `web/tests/`：覆盖"历史 URL 命中即丢""无历史期时行为不变""窗口边界（N 天外的不算）"；启用标题 hash 档时覆盖"同标题不同 URL 命中"。

## 实现要点

- **历史窗口**：默认 **7 天**（可配 N）。从 `web/runs/issues/<issue_id>.json` 读最近 N 期的头条+速览，取其 `url` 集合（第二档再取归一化标题 hash 集合）。**排除当期自身**；按**日期窗口**取，不假设每日连续（已有空档，如 26→29）。
- **URL 去重（核心、必做）**：候选 `url` ∈ 历史 URL 集 → 直接丢。建议比对前对 URL 做轻量 canonical 化（去常见 tracking 参数、统一 http/https、去尾斜杠），但**保持简单**，别过度。URL 完全相同才丢——**零误判**，这是最稳的一刀。
- **标题归一化 hash（可选第二档）**：小写 → 去标点 → 去媒体前缀 → 去常见噪声词（英 `reportedly/says/launches/announces…`、中 `独家/首发/据悉…`）→ hash；命中历史标题 hash 集 → 丢。噪声词表进 config 便于增补。这一档默认可开，仍零依赖、几乎不误判。
- **就这两档，到此为止**：不做字符相似度、不做事件指纹、不做 `near_duplicate`/`possible_followup` 分级、不做 penalty、不做跟进框定。这些在 [spec §7.5](../spec/01-科技日报.md) 标了"后续"，本工单不碰。
- **不必新增 pipeline 阶段**：优先把这道过滤**并进本地预筛**（复用 `seen_urls`），footprint 越小越好；除非并入确实别扭，否则不要为它单开 `PIPELINE_STAGES` 条目。
- **向后兼容**：无历史期 / 关掉开关 / config 缺省时安全降级，行为同今天（不滤任何东西）。

## 验收标准

- 构造两期已发日报落进 `web/runs/issues/`，再跑一批候选过本地预筛：
  - 候选里**与近 N 天已发同 URL** 的 → 被丢、不进 AI 粗筛；
  - URL 不在历史集里的 → 正常保留；
  - 窗口外（> N 天）的旧期 URL **不**参与去重。
- （启用标题 hash 档时）同标题、不同 URL 的候选 → 被丢。
- 无历史期或开关关闭时，pipeline 行为与今天一致。
- `pytest -q` 全过。

## 非目标

- 不做字符相似度（difflib / n-gram）、不做事件指纹、不做实体抽取。
- 不做 `near_duplicate` / `possible_followup` 分级、不做 penalty 降权、不做"跟进"框定。
- 不引入 AI / 不新增 AI task_type / 不引 rapidfuzz、jieba 等依赖。
- 不改 AI 选题/写作 prompt、不动阅读链路、前端、Supabase、feedback/profiles。
- 不做源失败容错、空板块处理（仍属 M3 其余部分）。

## 开放问题

- 上线跑一周后，若仍有"换源/换稿"的同事件重复（URL 与标题都不同）——再评估是否补相似度/指纹那一档（即原计划的完整版），届时另开工单。
