## 目标

解决 Codex 接管 `issue_compose` 后出现的翻译腔、抽象名词堆叠和机械句式。先用 2026-07-22 同一批候选与选题做离线盲测，再由用户决定正式管道保留单阶段还是增加独立中文编辑阶段。

## 产品决策

1. `humanizer-zh` 只作为规则研究参考，不成为生产依赖；生产使用项目内固定版的“中文新闻编辑规则”。
2. A 为已发布的 7 月 22 日成稿；B 在 compose prompt 中加入专用规则与真实正反例；C 从 A 的事实稿出发，再做一次只改表达的中文编辑。
3. 首轮只生成私有实验产物：不改 `05_issue.json`，不写 `web/issues`、`web/dist`，不触发发布。
4. C 只能修改标题、摘要、头条精读正文和影响分析。ID、来源、分数、顺序、数量、栏目标签、引语、丢弃项与合并记录全部锁定。
5. 结构、数字或主体守卫失败时，C 标记为无效并回退到 A；实验命令本身不会把 C 接入发布链路。
6. 用户最终验收混合方案 D：事实稿决定内容边界，中文编辑可以拆句、合句、重排和去重，但不得改变事实程度或引入未采用细节。
7. 正式管道保留 `05_issue_draft.json`，D 编辑后的 `05_issue.json` 进入发布。单条守卫失败只回退该条；AI 整体调用失败时整期回退事实稿。
8. D 从后续新生成的日报开始生效，不重生成或覆盖 2026-07-22 及其他已发布日报。

## 实现范围

- 新增项目内固定的中文新闻编辑规则与 Golden Set 正反例。
- 为 `build_issue_file_prompt` 增加可选的中文规则注入，不改现有默认行为。
- 新增 `issue_humanize` 任务类型、二次编辑 prompt 与本地守卫。
- 新增离线评测命令，在原 run 下的独立 `zh-editor-eval/` 目录保存 B、C、校验报告、盲评稿和方案映射。
- 盲评稿覆盖 4 条头条和 6 条速览；头条展示标题、摘要和影响分析，速览展示标题和摘要。
- 新增 D 离线评测命令和按条目回退的事实守卫。
- 将 `issue_humanize` 接入 `issue_compose → issue_humanize → publish_frontend` 正式管道，同时保留独立手动命令与 provider 可替换边界。

## 验收标准

- 实验前后 7 月 22 日的 `05_issue.json`、`web/issues`、`web/dist` Git 差异为零。
- B/C 都使用同一份 7 月 22 日候选、选题和基线稿；盲评稿不暴露 A/B/C 映射。
- C 的可编辑字段以外完全一致；新数字、新英文/混合主体或关键中文主体会触发失败回退。
- 结构不变、字段不丢失，原有全套测试通过。
- 用户对自然度、易懂度、信息完整度和 AI 腔完成盲评后，再按 0.5 分门槛写入生产结论。
- 正式 D 阶段产物可追溯，单条校验失败只回退对应条目，AI 调用失败时整期安全回退。

## 生产决策结论

- B 语言得分最高，但事实复审不通过，淘汰。
- C 事实更稳，但改写自由度不足。
- D 结合 B 的中文重组能力与 C 的事实边界，已由用户完成语言与信息保留验收，作为正式生产方案。

## 混合方案 D 验收补充

- D 可以拆句、合句、重排和去重，不锁句子数或要求字面最小变更。
- A 定义成稿的事实范围；候选原文只用于核对，不借机扩写 A 没有选用的细节。
- 双向保留关键数字、标题主体和事实限定词；不得把“已成功”改成“试图”，或反向升格。
- 某条校验失败时只回退该条 A 原文；锁定字段或整体结构改变时才整期回退。
- 2026-07-22 真数据 D 已生成，用时约 4 分 38 秒；修正两处过度机械的误报后，结构、数字、标题主体和限定词校验通过，未触发条目回退。
- D 已修复 B 的两个已知风险：精读明确保留“从生产数据库取得测试答案”；Buzz 速览未再新增用户规模断言。
- 独立语义审计调用因 Codex 严格 schema 命令异常未执行，不记为通过；已人工回对 A/D 差异，用户已验收 D 的语言与信息保留表现。
