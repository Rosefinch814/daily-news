## 目标

> **2026-07-22 用户纠偏**：本工单原定的“AI 标题失败后固定日期标题仍可发布”已被 [T18](T18-xhs-strict-ai-export.md) 废止。正常发布导出的标题必须是 AI 产出且校验通过，失败则阻断导出。下文保留作为历史实现记录。

让 `export-xhs` 在出图的同时，产出**发小红书用的标题 + 正文**：标题走 AI 钩子（≤20 字、忠实、可回退），正文精简拼装（hashtag + 头条 3 条 + slogan，逐字复制、零漂移）。行为权威见 [spec/小红书图组导出 §9](../spec/小红书图组导出.md)。

## 范围

1. **标题（AI 钩子 + 兜底）**：新增标题生成，走**可替换 `ai_engine` 入口**（复用 T13 的 provider/日志模式），挑当天最有料角度写一句 **≤20 字** 标题；AI 不可用 / 超 20 字 / 跑偏 → **回退固定格式**「AI科技日报 · M月D日」。
2. **正文（精简拼装 · 零 AI）**：hashtag 行 + 今日头条 3 条标题（编号、**逐字复制**）+ 一句 slogan；**速览不逐条列**；目标 ~150–250 字，硬顶 ≤1000 字。
3. **产出**：整合进现有 `caption.txt`（标题首行、空行、正文），或独立 `title.txt`/`body.txt`——实现定，完成说明写明。
4. hashtag / slogan / 刊名走**配置常量**（复用 T12 的 `XHS_PUBLICATION_NAME` 同源），一处改。

## 涉及文件

- `web/src/daily_news/ai_engine.py`（或 T13 落 `xhs_condense` 处）：加标题钩子 AI 任务 + 文件版 prompt 构造器，复用现有 `run_ai_task`/`save_ai_task_run` 与 provider 选择。

- 只动 `web/`。

## 实现要点

- **标题 prompt 铁律（写进模板 + 落测试）**：≤20 字；**忠实、不标题党、不新增/不改写事实**（呼应 spec §3.3）；可用当天头条角度，但不得编造未发生的说法。prompt 是**写死模板**、AI 只填内容不改模板（全项目安全边界：AI 不重写自己的指令）。
- **标题硬校验 + 回退**：AI 返回后**程序侧再数一遍字数**，>20 或空或异常 → 回退固定格式，**永不**让超限标题漏出。记一行日志。
- **正文逐字复制**：头条 3 条标题直接取 `issue.headlines[:3].title_zh`，**不经 AI、不改写**；hashtag/slogan 为常量。整体拼完做一次 ≤1000 字兜底（超则先砍到少列，仍超再截到完整句）。
- **`--no-ai-condense` 一致性**：T13 已有关闭 AI 的开关；标题钩子应受**同一开关**（或类似开关）控制——关掉时标题走固定格式，保证纯确定性导出可复现。
- 复用 T13 已建的 AI 优先 / 兜底回退骨架，别另造一套。

## 验收标准

- 跑 `export-xhs`：产出**标题**（≤20 字、读着忠实不标题党）+ **正文**（hashtag + 头条 3 + slogan、~150–250 字、≤1000）。
- 头条 3 条标题与 issue 逐字一致（无改写、无漂移）。
- 关掉 AI（`--no-ai-condense` 或等价）：标题走固定格式「AI科技日报 · M月D日」，导出仍成功、可复现。
- provider 不可用时不崩、标题回退固定格式。
- 切 provider 不改导出业务逻辑。
- `pytest -q` 全过。

## 非目标

- 不做自动发布小红书（导出文案，仍手动发）。
- 不改卡片视觉/分页（T12）与正文收敛（T13）。
