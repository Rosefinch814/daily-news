## 目标

只让 `export-xhs --cover-template v2` 的**封面大字**从"头条 1 标题只压不改"升级为"头条 1 标题**磁化**"——在反漂移铁律内把新闻标题说得**更想让人点**（feed 第一眼的点击率杠杆）。一次产**两档**：克制版（默认上 v2 封面）+ 冲版（存产物、不发，供用户 A/B）。行为权威见 [spec §4.2](../spec/小红书图组导出.md)。

> **版本隔离是铁律**：`xhs_magnetize` 只对 `v2` 生效。`classic` 和 `single-hook` 不调用磁化任务，封面文案、视觉、目录和其他行为完全不变。视觉版式归 T16，本工单不碰版式。

## 范围

1. **新增 AI 任务 `xhs_magnetize`**（走可替换 `ai_engine` 入口，复用 T13 的 provider 选择 / 日志 / `save_ai_task_run` 骨架，**别另造**）：输入 = 头条 1 的 `{title_zh, summary_zh}`；输出 = `{restrained, punchy}` 两条磁化标题。
2. **v2 封面大字来源改为 `restrained`（克制版）**；`punchy`（冲版）写进产物供 A/B（见范围 4）。v2 原 `cover_hook`（头条 1 标题收敛）**保留为兜底**（见护栏）。
3. **反漂移硬校验（程序侧，AI 之外再验一遍）**——见"实现要点"，任一不过 → **回退 `cover_hook` 收敛**（原标题只压不改）。
4. **冲版落产物**：把 `punchy`（及原标题、克制版）写进 v2 专属的 `web/runs/xhs/<date>-v2/` 一个文件（如 `cover_title_variants.txt` 或并入现有产物），标注哪条在用、哪条备选，方便用户手动换图测试。
5. **严格触发条件**：仅当 `cover_template == "v2"` 时调用 `xhs_magnetize`、执行磁化校验并写变体产物；其他模板连该 AI 任务都不触发。
6. 磁化区间沿用 [spec §7](../spec/小红书图组导出.md) 的 v2 封面大字档；v2 字号降档逻辑不变（T16 已有）。

## 涉及文件

- `web/src/daily_news/xhs_export.py`：仅在 v2 分支中把封面大字取值改为 `restrained`；接 `xhs_magnetize` + 硬校验 + 回退 `cover_hook`；写冲版产物。
- `web/src/daily_news/ai_engine.py`（或 T13 落 `xhs_condense` 处）：加 `xhs_magnetize` 的**写死 prompt 模板**（含下方 few-shot 正负例）+ schema `{restrained, punchy}`。
- `web/config/pipeline.yaml`（或对应 stage_providers）：给 `xhs_magnetize` 配 provider（默认同其它 AI 阶段）。
- 情绪词表、禁改项等落**可配置常量**（不散写魔法值）。
- `web/tests/test_xhs_export.py`：覆盖下方验收。
- 只动 `web/`。

## 实现要点（护栏 = 复现质量的关键）

- **prompt 写死、AI 只填不改模板**（全项目安全边界）。模板须含：事实层/表述层的线 + 4 个合法动作 + 下方 few-shot 正负例。
- **硬校验（程序侧，落测试）**：
  - **数字校验**：`restrained`/`punchy` 里出现的数字/金额/百分比，必须都能在 `title_zh + summary_zh` 里找到（归一化后子串/集合比对）；出现新数字或改数字 → 违规。
  - **主体校验**：主体公司/人物不得替换或杜撰（至少校验原标题主体仍在、未被换成别的实体）。
  - **克制版禁用情绪词表**（可配常量：`震惊/太火了/暴涨/崩盘/吊打/秒杀/逆天…`）——`restrained` 命中即违规（`punchy` 允许有限情绪，但仍过数字/主体校验）。
  - **长度**：落 spec §7 封面大字区间；超长按收敛规则收完整句、无省略号。
  - **违规处理**：`restrained` 不过 → 封面大字**回退 `cover_hook` 收敛**（原标题只压不改）；`punchy` 不过 → 冲版记为"无"、不影响出图。记一行日志。
- **`--no-ai-condense` 一致性**：关掉 AI → v2 封面大字 = `cover_hook` 收敛，不磁化，确定性可复现；旧模板依旧走各自现有逻辑。
- provider 挂了/超时 → 不崩，回退 `cover_hook`。
- **旧模板回归锁定**：为 `classic` / `single-hook` 注入可计数的假 magnetizer，两者导出后调用次数必须为 0，且关键封面文案与 T17 前完全一致。

## few-shot 正负例（塞进 prompt 模板 · 用户已校准）

**格式**：原标题 → 克制版（默认）｜ 冲版（备选）。事实层锁死，只动表述层。

- `谷歌自研新芯片Frozen v2曝光，目标2028年让Gemini能效提升6至10倍`
  → 克制「谷歌自研新芯片曝光，能效目标提升6至10倍」｜ 冲「6到10倍！谷歌要给Gemini换颗自研芯」（"目标"不得丢成"已实现"）
- `国行苹果智能等7款手机端侧AI获备案，三星把端侧大脑交给国产面壁MiniCPM`
  → 克制「苹果等7款端侧AI获备案，国行AI要来了」｜ 冲「三星把手机AI大脑，交给了国产面壁」（"获备案"不得说成"已上线"）
- `Anthropic 15亿美元版权和解获终审批准，每部赔3000美元、覆盖约50万部作品`
  → 克制「Anthropic 15亿美元版权和解获批，每部赔3000美元」｜ 冲「美国版权史最大和解：Anthropic赔15亿」（"史上最大"须原文有据）
- `黄仁勋两天扫遍日本产业界：万亿日元押注主权"物理AI"，底座仍是英伟达芯片`
  → 克制「黄仁勋两天扫遍日本，万亿日元押注"物理AI"」｜ 冲「黄仁勋两天扫遍日本，砸下万亿日元」
- `Kimi K3发布48小时算力告急：月之暗面暂停新订阅，评测逼近前沿引芯片股回调`
  → 克制「Kimi K3发布48小时算力告急，月之暗面暂停新订阅」｜ 冲「太火了！Kimi K3发布48小时，算力就顶不住」（"逼近"不得写成"超过"）
- `智谱 ARR 半年增长 15 倍达 10 亿美元，追平 Anthropic 的代码变现速度`
  → 克制「智谱ARR半年增长15倍，达10亿美元」｜ 冲「半年翻15倍，智谱ARR冲到10亿美元」

**反例（禁止，越事实层）**：
- ❌「智谱**估值**暴涨15倍」——ARR 偷换成估值（改事实）。
- ❌「**震惊！**智谱ARR暴涨」——震惊体 + "暴涨"程度夸大。
- ❌「谷歌芯片**吊打英伟达**」——无中生有的对比/指控。
- ❌ 任何新增原文没有的数字、主体、时间、因果。

## 验收标准

- 跑 `export-xhs --cover-template v2`：封面大字为**克制版磁化标题**（比原标题更抓眼、但数字/主体/程度与原文一致）；`<date>-v2/` 产物里能看到**冲版**备选 + 原标题。
- 跑 `export-xhs --cover-template classic` 和 `single-hook`：不调用 `xhs_magnetize`、不生成 `cover_title_variants`，原有封面文案与目录完全不变。
- 硬校验生效：构造"磁化输出新增了原文没有的数字 / 换了主体 / 克制版含情绪词"的用例 → **回退原标题收敛**、不发违规文案。
- 关掉 AI（`--no-ai-condense`）：封面大字 = 原标题收敛，导出成功、可复现。
- provider 不可用不崩、回退兜底。
- 切 provider 不改导出业务逻辑。
- `pytest -q` 全过。

## 非目标

- 不改视觉版式（T16）、不改支撑句/速览收敛（T13 既有槽位）、不改 caption 标题正文（T14）。
- 不对 `classic` / `single-hook` 做磁化，不会把 v2 的文案逻辑下沉为所有封面共享默认。
- 不做自动 A/B 或自动发布（冲版只是"存下来供人手动测"）。
- 不磁化头条页/速览页标题、不磁化日报正文（专业简报体不动）。
