# MVP —— 科技单板块

> 会逐步更新。MVP 的重点是**验证**，不是打磨。

## 一句话

科技单板块：从几个科技源抓取 → 用 Claude Code 按"关注清单"筛选 + 摘要 + 翻译 → 生成一期 HTML 电子报 → 末尾带反馈入口，能调教下一期。

## 核心用户任务

让用户每天打开一份**只含他想看的科技内容**的电子报，并能通过反馈让它越来越懂自己。

## 最大技术风险（MVP 要先证明的）

**AI 能否按"关注清单 + 口味档案"筛出"我想看的"，并随反馈变准。**
——这是整个产品的命门。抓取/渲染都是成熟技术，真正不确定的是"精准 + 可调教"。

## 最小流程（同时证明价值 + 风险）

1. 配科技板块：3–5 个源 + 一份种子关注清单 + 一个空白口味档案。
2. 手动跑一个命令 → 生成当天 HTML 电子报（含摘要、英文翻译、相关度）。
3. 电子报末尾有最简反馈入口（👍/👎 或一句话）。
4. 给一次反馈 → 重跑 → 看下一期是否朝偏好偏移。

## 科技板块配置（种子）

**口味/重心**：全景科技——AI/大模型、芯片/硬件、大公司动态/产业格局、产品/消费科技。
广覆盖靠源保证，精准靠关注清单 + 口味档案调教收敛。

**数据源**（优先 RSS/API、不碰付费墙）：
- 英文：The Verge、Ars Technica、Reuters Technology、TechCrunch、Hacker News（科技综合）
- 中文：36氪、爱范儿

**关注清单种子**（口味档案的第一颗种子，会随反馈生长）：
```yaml
companies: [英伟达, 英特尔, 苹果, 微软, 谷歌, 特斯拉, 台积电, 阿里, 腾讯]
themes: [AI产品发布, AI芯片, 半导体, 大模型进展, 自动驾驶, 消费电子新品]
```

## 验收标准

1. **精准**：读完认为"基本是我想看的科技内容"，没有明显"该来没来 / 不该来却刷屏"。
2. **调教**：给一条"多看 X / 少看 Y"反馈后，下一期筛选可见地朝偏好偏移。

## 明确推迟（非目标）

财经/其他板块、定时调度、邮件推送、精美样式、跨源去重、源容错、多板块 UI、多用户。
