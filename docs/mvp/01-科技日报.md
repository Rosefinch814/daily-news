# MVP —— 01·科技日报

> **本期（01·科技日报）的源头文档 = 目标。** 定义这一期要达成什么、怎么算成功。
> 完整功能与方案见 [spec](../spec/01-科技日报.md)；做事步骤见 [里程碑](../milestones/01-科技日报.md)。

## 一句话

一份科技板块的个性化电子报：抓权威源 → AI 按你的关注清单筛选分层 → 全中文摘要/精读 →（M2 起：按编号反馈 → 越来越懂你）。

## 核心用户任务

每天打开一份**只含我想看的科技内容、全程中文读得懂、还帮我点透影响**的电子报。（让它"越来越懂我"的反馈调教是下一步＝M2。）

## 要证明什么（最大技术风险）

**AI 能否仅凭「关注清单」就从 7 个源里筛出"我想看的"，并全程中文可读。**

抓取/渲染是成熟技术；真正不确定的是 **精准筛选**——v1 先把这一半验证扎实。"随反馈越来越准"（调教）是更难的下半段，放 M2。

## 本期范围边界（高层）

**做（v1＝精简验证版，只留能证明"精准筛选 + 全中文"的核心）：**
- 科技**单板块**（多板块架构先只落一个），数据源**全部 7 个**（见 spec §13）
- **报头 + 头条精读 + 速览** 两层（手动生成 ≈ 日报）
- **全程中文**（英文源翻译，原文仅备查）
- **AI 筛选分层**（只读关注清单打双分 → 丢弃/速览/头条；同源合并）

**不做（本期不做，归 M2）：**
- **调教闭环整套**：反馈区 + 口味档案 + 消化反馈（产品灵魂，但 v1 先不做；v1 筛选只靠关注清单）
- 📝 今日导读、📡 关于本期（源面板）、← 上下期/归档导航 → ——"报纸外壳"，推到 M2

**不做（更后期非目标）：**
- 财经/其他板块、定时自动生成、点击式 👍👎/本地服务、独立去重系统、源失败容错、精美视觉打磨、多板块管理 UI、多用户

> 具体**怎么做**（版面、卡片、关注清单、筛选、内容生成规范、数据源等）→ 全部见 [spec](../spec/01-科技日报.md)。

## 验收标准（怎么算达成目标）

1. **精准**：读完认为"基本是我想看的科技内容"，无明显"该来没来 / 不该来却刷屏"。
2. **全中文可读**：每条都能纯中文读懂，无需打开英文原文。

> "调教"（给反馈后下一期朝偏好偏移）是 **M2 的验收标准**，v1 不验。
