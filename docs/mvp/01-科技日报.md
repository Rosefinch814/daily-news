# MVP —— 01·科技日报

> **本期（01·科技日报）的源头文档。** 本文定义这一期要实现的功能；
> 同期的 [spec](../spec/01-科技日报.md) 与 [里程碑](../milestones/01-科技日报.md) 都**基于本 MVP 派生**。
> MVP 的重点是**验证**，不是打磨，后续逐步优化。

## 一句话

一份科技板块的个性化电子报：抓权威源 → AI 按「关注清单 + 口味档案」筛选分层 → 全中文摘要/精读 → 生成 HTML 报纸 → 你按编号反馈 → 调教下一期，越来越懂你。

## 核心用户任务

每天打开一份**只含我想看的科技内容、全程中文读得懂、还帮我点透影响**的电子报，并能通过反馈让它越来越懂我。

## 最大技术风险（MVP 要先证明）

**AI 能否按「关注清单 + 口味档案」筛出"我想看的"，并随反馈变准。**
抓取/渲染都是成熟技术；真正不确定、决定成败的是 **精准 + 可调教**。

## 底层原则（已锁）

1. **单板块**：多板块架构，MVP 只建"科技"一个板块。
2. **含调教**：MVP 就包含最小调教闭环，体现"越来越懂我"。
3. **🔑 全程中文铁律**：绝不需要读英文原文；英文源一律由 AI 给中文，原文链接只作备查。
4. **🚫 忠于原文红线**：摘要/精读/翻译只能基于原文事实，AI 不许加料、不许脑补数字与细节。
5. **节奏**：手动生成 ≈ 日报；每期分层：头条精读 + 速览。

## A. 一期版面

```
报头（我的日报·科技 / 日期·期号）
  ↓
📝 今日导读（AI 一段当天综述，先给全局）
  ↓
⭐ 头条精读 3-5 条（编号 头条1…）
  ↓
⚡ 速览 10-15 条（编号 速览1…）
  ↓
📡 关于本期（抓了几源/几条 → 留几条，透明建信任）
  ↓
💬 反馈区（按编号点评 + 一句话"多看/少看"）
  ↓
← 上一期 · 📚 归档 · 下一期 →
```

## B. 卡片字段

| 字段 | 速览 | 头条 |
|---|---|---|
| 编号 | ✅ | ✅ |
| 中文标题 | ✅ | ✅ |
| 中文摘要 | ✅ 短(2-3句) | ✅ |
| 〔展开〕中文精读 | ✅ 可点开（按需生成） | ✅ 默认更深 |
| 📊 影响（AI 分析，标注） | — | ✅ |
| 💡 为什么推给你 | — | ✅ |
| 来源 · 原文↗（备查） | ✅ | ✅ |

**头条分块结构**（事实与 AI 判断泾渭分明）：
```
[头条1] 中文标题
摘要：一两句把事说清。（事实）
精读：来龙去脉讲透——发生了什么、关键信息、背景。（事实，忠于原文）
📊 影响（AI 分析）：对竞争格局/你关注的公司可能意味着…（← 标清是 AI 判断）
💡 为什么给你：命中关注「英伟达 / AI芯片」+ 行业大事
来源 综合 The Verge、Reuters · 原文↗
```

## C. 关注清单（你手写的种子）

```yaml
想看:
  公司: [英伟达, 苹果, 台积电, 特斯拉, 微软, 谷歌, 英特尔]
  主题: [AI芯片, 大模型进展, AI产品发布, 自动驾驶, 半导体]
  人物: [黄仁勋, 马斯克, 奥特曼, 库克]
不想看:                # 负向：命中即降权/丢弃，专滤噪音
  [小公司融资, 纯营销稿, 加密货币炒作, 谍照外观爆料]
```

## D. 调教闭环（灵魂）

- **反馈形式**：按**编号**的细点评（`头条2 👍 / 速览5 👎`）+ 一句话方向（`多看芯片，少看外观`）。
- **回传方式（MVP）**：**方案 A 纯文本**——你把反馈写给下次生成（编辑文件/喂给脚本）；**网页不放点击按钮**（无后端，点了没人接）。以后再升级本地小服务让 👍👎 可点。
- **口味档案**：一份活文档（每板块一份），AI 把反馈消化成**偏好规则**（如"对英伟达兴趣最高""少看融资/营销稿"）。
- **如何生效**：下次筛选时 AI **同读「关注清单 + 口味档案」**打分 → 越用越准。

## E. 筛选逻辑（精准在这发生）

```
每条新闻 → AI 拿「关注清单 + 口味档案」打两个分：
   · 和你的相关度    · 事件重要度
        ↓
命中"不想看" / 太不相关 → 丢弃
相关                    → 速览
相关 且 重要            → 头条（3-5 名额，两分都高者胜出）
```
- **同源合并**：同一事件多源都报时，AI 在筛选时顺手捏成一条（标"综合 X、Y"）。MVP 不做独立去重系统。

## 内容生成规范

| 任务 | 规范 |
|---|---|
| 文风 | **专业简报体**（信息密度高、精炼专业，像高端 newsletter） |
| 筛选打分 | 相关度 + 重要度双分；命中负向清单降权 |
| 中文摘要 | 每条 2-3 句，忠于原文 |
| 头条精读 | 一段把事讲透（事实）+ 标注的「📊影响」分析 |
| 为什么推给你 | 一句点明命中的关注点 |
| 翻译 | 英文源融进中文摘要/精读，原文仅备查 |
| 今日导读 | 一段综述当天最值得注意的，先给全局 |
| 红线 | 全程忠于原文，AI 判断必须标注，不编造 |

## 科技板块配置（种子）

**重心**：全景科技——AI/大模型、芯片/硬件、大公司动态/产业格局、产品/消费科技。
广覆盖靠源，精准靠关注清单 + 口味档案调教收敛。

**数据源**（优先 RSS/API、不碰付费墙）：
- 英文：The Verge、Ars Technica、Reuters Technology、TechCrunch、Hacker News（科技综合）
- 中文：36氪、爱范儿

## 验收标准

1. **精准**：读完认为"基本是我想看的科技内容"，没有明显"该来没来 / 不该来却刷屏"。
2. **全中文可读**：每条都能纯中文读懂，无需打开英文原文。
3. **调教**：给一条"多看 X / 少看 Y"反馈后，下一期筛选可见地朝偏好偏移。

## 非目标（MVP 不做）

财经/其他板块、定时自动生成、点击式 👍👎 / 本地服务、独立去重系统、源失败容错、精美视觉打磨、多板块管理 UI、多用户。
