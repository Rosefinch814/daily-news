# MVP —— 01·科技日报

> **本期（01·科技日报）的源头文档 = 目标。** 定义这一期要达成什么、怎么算成功。
> 完整功能与方案见 [spec](../spec/01-科技日报.md)；做事步骤见 [里程碑](../milestones/01-科技日报.md)。

## 一句话

一份科技板块的个性化电子报：抓权威源 → AI 按你的关注筛选分层 → 全中文摘要/精读 → 你按编号反馈 → 越来越懂你。

## 核心用户任务

每天打开一份**只含我想看的科技内容、全程中文读得懂、还帮我点透影响**的电子报，并能通过反馈让它越来越懂我。

## 要证明什么（最大技术风险）

**AI 能否按「关注清单 + 口味档案」筛出"我想看的"，并随反馈变准。**

抓取/渲染是成熟技术；真正不确定、决定成败的是 **精准 + 可调教**——这是本期要验证的核心。

## 本期范围边界（高层）

**做：**
- 科技**单板块**（多板块架构先只落一个）
- **含最小调教闭环**（体现"越来越懂我"）
- **手动生成** ≈ 日报，分层：头条精读 + 速览
- **全程中文**（英文源翻译，原文仅备查）

**不做（非目标）：**
- 财经/其他板块、定时自动生成、点击式 👍👎/本地服务、独立去重系统、源失败容错、精美视觉打磨、多板块管理 UI、多用户

> 具体**怎么做**（版面、卡片、关注清单、筛选、内容生成规范、数据源等）→ 全部见 [spec](../spec/01-科技日报.md)。

## 验收标准（怎么算达成目标）

1. **精准**：读完认为"基本是我想看的科技内容"，无明显"该来没来 / 不该来却刷屏"。
2. **全中文可读**：每条都能纯中文读懂，无需打开英文原文。
3. **调教**：给一条"多看 X / 少看 Y"反馈后，下一期筛选可见地朝偏好偏移。
