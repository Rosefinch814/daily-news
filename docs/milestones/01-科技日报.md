# 里程碑 —— 01·科技日报

> **本期（01·科技日报）里程碑，基于 [MVP](../mvp/01-科技日报.md) 拆解。**

## M1 —— MVP：科技日报，验证"筛得准 + 全中文"（最高优先）

**范围（v1＝精简验证版）**
- 只做科技 1 个板块：**全部 7 个源**（spec §13）+ 种子关注清单（公司/主题/人物/不想看）。
- 手动跑命令 → 生成当天 HTML 电子报：**报头、头条精读(3-5，分块：摘要/精读/📊影响)、速览(10-15)**。
- 全中文（英文源翻译）；筛选**只读关注清单**、相关度+重要度双分、同源合并；文风专业简报体。
- 不含调教闭环（反馈区/口味档案/消化反馈）、今日导读、关于本期源面板、归档导航——全归 M2。

**验收标准**
1. **精准**：读完"基本是我想看的"，无明显该来没来/不该来却刷屏。
2. **全中文可读**：每条纯中文读懂，无需开英文原文。

**当前进度（2026-06-30）**

_设计 / 规格（已定稿）_
- v1 设计基调 V1a「报纸感」定稿；[design/设计规范.md](../../design/设计规范.md) 固化为样式唯一权威（token/字体/组件/响应式）。
- **高保真原型 [design/prototype/v1-科技日报.html](../../design/prototype/v1-科技日报.html) 完成**，作为 v1 渲染目标（范围＝报头＋头条精读＋速览）；落地形式＝前后端分离（客户端渲染，见下）。
- 交互/版面回流进 spec §4/§5/§14：`<details>` 手风琴精读、速览不给精读（两栏密排简讯栏列表）、头版合并为单一事实摘要。
- 报头定稿：刊名 **Tourbillion News** ＋ 英文板块副标题 **Technology**；去掉报眉、期号、「全程中文」标语，签名朱红落在刊名中缀点。
- spec §12 技术栈已定：Python ＋ feedparser/trafilatura/Jinja2/PyYAML/requests，`ai_engine(task,payload)->dict` 单一可替换入口，五段式流水线（fetch/filter/extract/generate/render）产 JSON 制品。

_实现（MVP 主链路已完成）_
- 科技日报已经能完成从抓取、筛选、正文补全、AI 选题、AI 写作到前端发布数据导出的主流程。
- 阅读端已从整页静态 HTML 调整为「前后端分离 · 客户端渲染」：前端页面固定，日报内容由每期 JSON 数据驱动。
- 公网访问已接入 Cloudflare Pages，当前公开站点可由 GitHub 推送触发部署。
- 生成侧已支持手动分步执行，也已新增一键 pipeline；三个 AI 阶段可分别选择 Codex 或 Claude。
- 日志与阶段产物已分离，便于后续排查问题、打包日志和观察 AI 阶段稳定性。
- 阅读链路仍为静态站点；Supabase 仅用于反馈采集与本地消化，不承载日报内容读取。

_当前结论_
- M1 已完成并通过多日真实运行验证；后续不再以 M1 能否跑通为主要风险。
- 主要风险已转向连续运行质量：AI 输出稳定性、字数控制、重复新闻、反馈消化是否准确。

**非目标**
调教闭环（反馈/口味档案，→M2）、财经/其他板块、定时、点击式👍👎/本地服务、独立去重、源容错、精美样式、多板块 UI。

## M2 —— 调教闭环 + 可用主流程（产品灵魂在此落地）

**范围**

_调教闭环（核心，方案见 [spec §9](../spec/01-科技日报.md)）_——拆成可验收条目：
- **反馈采集（点击式）**：每条新闻后 👍/👎/补充说明 + 报纸末尾整期留言区；交互已定稿（入口＝常驻微缩图标），回流 [spec §14.1](../spec/01-科技日报.md) + 设计规范，参考原型 `design/prototype/m2-反馈交互-科技日报.html`。〔工单 T04，已就绪〕
- **存储（Supabase 轻后端）**：新增 `feedback` 表；前端用 **anon key + RLS（仅匿名 INSERT）** 直接写库，service key 只留服务端；不导出文件。〔T03〕
- **消化（digest_feedback）**：可独立 `digest-feedback` 命令、也作 run-pipeline 起手阶段；攒几天批量、按 `issue_date` 对回新闻，**按类型分流**增量更新三档档案（taste / style / seed-suggestions），消化后标记。〔T05〕
- **生效（注入）**：粗筛/选题每次注入 `taste.md`，写作每次注入 `style.md`；关注清单仍是硬边界，档案是软偏好。〔T06 / T07〕
- **安全边界**：反馈只喂偏好档案，AI 不重写 prompt 模板；种子关注清单仅出建议、用户确认才改。

_报纸外壳（v1 未做的补齐）_
- **新增今日导读、"关于本期"源面板、历史归档 + index 索引页 + 上下期导航**。
- 源扩到 ~8-10 个优质源；报刊样式做耐看。

**验收**
1. **调教·选题**：给一条"多看X/少看Y"反馈 → `digest-feedback` 后 `taste.md` 增量更新且旧偏好保留 → 下一期粗筛/选题可见地朝偏好偏移。
2. **调教·写作**：给一条"翻译生硬/太长"反馈 → `style.md` 更新 → 下一期写作风格可见变化。
3. **闭环安全**：反馈写入 Supabase 且前端产物不含 service key；种子清单未被自动篡改（仅出现在 seed-suggestions 待确认）。
4. 连续用几天，每天打开能舒服读完，且感觉"它在变懂我"。

**当前进度（2026-07-01）· 调教闭环已验收通过 ✅**
- T03–T07 全部完成：反馈采集（前端 anon key 直写 Supabase）→ digest 消化（taste/style/seed-suggestions 增量更新、旧偏好保留）→ 注入（选题读 taste、写作读 style）闭环全通。
- **效果验收通过（用户 2026-07-01 人工验收）**：真实喂反馈 → digest → 出下一期，选题/写作朝偏好可见偏移，验收 1/2/4 达成；闭环安全（前端产物无 service key、sections.yaml 未被自动篡改）验收 3 达成。
- 运行策略：`run-pipeline` 生成、`digest-feedback` 独立消化，两者解耦，避免生成链路被消化失败拖垮。

**⏳ M2 剩项**：
- 报纸外壳（非核心、无工单）：今日导读、"关于本期"源面板尚未做（历史归档/上下期导航已由 issue picker 覆盖、源已扩到 10）。
- **反馈两通道（公开后新增需求，T10/T11 todo）**：报纸要公开给别人读，故反馈拆为"我的口味反馈（私有喂 digest）"vs"读者反馈（公开留言箱 `product_feedback`，绝不喂 digest）"；靠 `feedback.owner_token` + digest 只认令牌保证"只有我能改我的报纸"。定义见 [spec §9.6](../spec/01-科技日报.md)。**修掉当前隐患：公开 `dist` 现带 anon key 且 digest 吃全部反馈，等于谁点都能改我的档案。**

**非目标**：定时自动化（cron，→M4）、推送、其他板块、反馈可视化分析面板、点完即生效的实时调教。

## M3 —— 持久化 & 边界

**范围**：跨源去重（独立、更强）、源失败容错、空板块处理、按日期归档回看、口味档案版本可追溯。

**验收**：个别源挂掉不影响整体；重复新闻不刷屏；档案演变可回看。

**当前进度（2026-07-01）**
- 跨期去重已落地**两层**：
  - T08：本地预筛硬去重——近 7 天历史 URL 去重 + 可选标题归一化 hash（零 AI、零依赖、零新增阶段）。
  - T09：选题级软去重——AI 读近 3 天已发极简索引判"换源转述/聚合/无新增事实"，换源不上头条、有新增事实按跟进；预筛对聚合稿加重降权封顶。**只影响选题分层，绝不本地硬删。**
- T09 由 Codex 先行实现，用户已人工验证 2026-07-01 期效果并认可，产品侧补 T09 追认。
- 更重的字符相似度 / 事件指纹分级仍未做——判重现由"已发索引 + AI 判断"承担，若发现漏网再另开工单。
- **源失败容错**：抓取已用 `asyncio.gather(return_exceptions=True)` 做到单源挂掉不拖垮整期（失败源标 `fetch_status=failed` 并降权），基本已具备。
- **空板块处理**：手动本地阶段由用户当场兜底，暂缓；待 M4 上定时无人值守时再一并处理（届时是其前置）。
- **口味档案版本可追溯**：方案已定＝用 git 管 `web/profiles/`（每次 digest 后提交，天然可 diff/回退/回看，无需另写快照）。**当前搁置**：`web/profiles/` 仍在 `.gitignore` 中未入库，因 repo 公开/私有未定（taste/style 含个人偏好，入公开库即公开）；用户定了 repo 可见性后再 un-ignore 落地。

**非目标**：自动化、视觉大改。

## M4 —— 自动化 & 打磨
