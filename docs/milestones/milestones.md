# 里程碑

> 会逐步更新。每个里程碑含：范围 / 验收标准 / 非目标。完整 MVP 方案见 [../mvp/mvp.md](../mvp/mvp.md)。

## M1 —— MVP：科技日报，验证"筛得准 + 全中文 + 能调教"（最高优先）

**范围**
- 只做科技 1 个板块：3-5 个源 + 种子关注清单（公司/主题/人物/不想看）+ 空白口味档案。
- 手动跑命令 → 生成当天 HTML 电子报：今日导读、头条精读(3-5，分块：摘要/精读/📊影响/💡为什么)、速览(10-15)、关于本期、反馈区、归档导航。
- 全中文（英文源翻译）；筛选用相关度+重要度双分、同源合并；文风专业简报体。
- 最小调教闭环：按编号文字反馈（方案A）→ AI 消化进口味档案 → 下次生效。

**验收标准**
1. **精准**：读完"基本是我想看的"，无明显该来没来/不该来却刷屏。
2. **全中文可读**：每条纯中文读懂，无需开英文原文。
3. **调教**：给一条"多看X/少看Y"反馈后，下一期可见地朝偏好偏移。

**非目标**
财经/其他板块、定时、点击式👍👎/本地服务、独立去重、源容错、精美样式、多板块 UI。

---

## M2 —— 可用主流程

**范围**
- 调优关注清单 + 筛选/生成 prompt + 口味档案机制。
- 源扩到 ~8-10 个优质源。
- 报刊样式做耐看；历史归档 + index 索引页。
- "关于本期"源面板 + 编辑导读打磨。

**验收**：连续用几天，每天打开能舒服读完，且感觉"它在变懂我"。

**非目标**：定时自动化、推送、其他板块。

---

## M3 —— 持久化 & 边界

**范围**：跨源去重（独立、更强）、源失败容错、空板块处理、按日期归档回看、口味档案版本可追溯。

**验收**：个别源挂掉不影响整体；重复新闻不刷屏；档案演变可回看。

**非目标**：自动化、视觉大改。

---

## M4 —— 自动化 & 打磨

**范围**：launchd/cron 每日自动生成；可选邮件/推送；升级反馈为本地小服务(👍👎可点)；视觉打磨；加新板块便捷化（为复制"财经"铺路）。

**验收**：早上打开就有今天的报，加板块只需改配置。
