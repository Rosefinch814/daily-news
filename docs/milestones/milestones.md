# 里程碑

> 会逐步更新。每个里程碑含：范围 / 验收标准 / 非目标。

## M1 —— MVP：科技单板块，验证"筛得准 + 能调教"（最高优先）

**范围**
- 只做科技 1 个板块，配 3–5 个源 + 种子关注清单 + 空白口味档案。
- 手动跑 `main` → 生成当天 HTML 电子报（摘要、翻译、相关度）。
- 末尾带最简反馈入口（一句话 / 👍👎），下次生效。

**验收标准**
1. 读完认为"基本是我想看的科技内容"。
2. 给一次反馈后，下一期筛选可见地朝偏好偏移。

**非目标**
财经/其他板块、定时调度、精美样式、去重容错、多板块管理 UI。

---

## M2 —— 可用主流程

**范围**
- 调优关注清单 + prompt + 口味档案机制。
- 板块源扩到 ~8–10 个优质源。
- 报刊样式做耐看；历史归档 + index 索引页。
- "关于本期"源面板 + 编辑导读。

**验收标准**
连续用几天，每天打开就能舒服读完，且感觉"它在变懂我"。

**非目标**
定时自动化、推送、其他板块。

---

## M3 —— 持久化 & 边界

**范围**
- 跨源去重（同一新闻多源合并）。
- 源失败容错、空板块处理。
- 按日期归档可回看；口味档案版本可追溯。

**验收标准**
个别源挂掉不影响整体；重复新闻不刷屏；档案演变可回看。

**非目标**
自动化、视觉大改。

---

## M4 —— 自动化 & 打磨

**范围**
- launchd / cron 每日自动生成。
- 可选邮件 / 推送。
- 视觉打磨；加新板块的便捷配置（为复制"财经"板块铺路）。

**验收标准**
早上打开就有今天的报，加板块只需改配置。
