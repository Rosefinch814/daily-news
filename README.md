# 我的日报 · daily-news

一份**属于自己的个性化新闻报纸**：由我自己定义的若干**板块**组成，每个板块自配数据源与关注点，并通过反馈**不断被调教、越来越懂我**。

> 不是又一个新闻聚合器。核心是 **精准 + 越来越懂我**。

## 现状

- **M1 科技日报主链路已完成**：从 RSS 抓取、筛选、正文补全、AI 选题和写作，到前端静态站生成，可用一条 `run-pipeline` 命令串行完成。
- **M2 调教闭环已完成**：反馈可进入 Supabase，`digest-feedback` 独立消化为 taste/style 档案，下一期生成会读取这些偏好。
- **当前正式 AI 默认为 Codex**：粗筛、选题、事实稿写作和混合 D 中文编辑均经由同一可替换 `ai_engine` 调用。
- **M4 部分完成**：小红书图组与发布文案可独立导出；每日定时启动、Git 自动推送和小红书自动发布尚未实现。

## 当前运行方式

```bash
cd web
./.venv/bin/daily-news run-pipeline --section tech --date YYYY-MM-DD
```

这条命令会自动完成：

```text
抓取 → 本地预筛 → AI 粗筛 → 正文补全 → AI 选题
     → AI 事实稿 → 混合 D 中文编辑 → 事实守卫/回退 → 生成前端文件
```

当前还不是无人值守：命令需要人工启动，Git 提交/推送、Supabase 完整 run 同步、反馈消化、小红书导出与发布仍是独立步骤。

## 核心概念

```
报纸 = 板块的集合
板块 = 名字 + 数据源 + 关注清单(种子) + 口味档案(会长大) + 反馈入口
```

- **关注清单**：第一天的我（种子偏好）。
- **口味档案**：被调教后的我（活的偏好记忆，随反馈生长）。
- **调教**：不训练模型，靠"反馈 → 口味档案 → 越来越准"的记忆闭环。
