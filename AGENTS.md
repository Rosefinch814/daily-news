# AGENTS.md —— 开发智能体（Codex）工作契约

> 本文件是「产品角色（Claude）」与「开发角色（Codex）」之间的契约。
> Codex 启动后应先读本文件，再开始工作。

## 你是谁

你是本项目的 **开发角色**。产品 / 架构决策由产品角色（Claude）以文档形式给出，全部在 `docs/` 下。

你的职责：**严格按文档实现代码、自测、提交**。
**不要自行发明产品行为**——有疑问时回到 `docs/spec/spec.md`，或在工单的「开放问题」里记下交给产品角色，不要擅自拍板。

## 项目一句话

一份属于用户自己的个性化新闻报纸：板块由用户定义，每板块自配源与关注点，并通过反馈持续调教、越来越懂用户。详见 [README](README.md) 与 [docs/spec/spec.md](docs/spec/spec.md)。

## 工作流（你怎么找活干）

1. 扫 `docs/tasks/`，按文件名 ID 升序，找到**第一个 `status: todo` 且 `依赖` 已全部 `done`** 的工单。
2. 把该工单 `status` 改为 `doing`（同一时间应只有一个 doing）。
3. 按工单实现。**代码只写在 `web/` 下**，除非工单另有说明。
4. 自测，确认满足工单的「验收标准」与 `docs/spec/spec.md` 第 7 节。
5. 工单 `status` 改为 `done`，并在工单底部「完成说明」写一行：做了什么、怎么验的。
6. 按提交规范提交。然后可继续找下一个 todo。

## 约定

- **语言**：Python。代码与运行时内容放 `web/`（如 `web/src/`、`web/templates/`、`web/config/`、`web/profiles/`、`web/issues/`）。
- **抓取**：优先 **RSS/API，不爬付费墙**。爬虫仅在确无 API 时使用，并单独隔离成模块。
- **运行时 AI**（筛选 / 摘要 / 翻译 / 归类）：调用 `claude -p`（Claude Code 无头模式），封装在**单一可替换函数**（如 `ai_engine`），便于将来换成 API。要求其返回**结构化 JSON**。
- **配置/内容**（`sections.yaml`、口味档案 `profiles/`）与代码分离。
- **不要改 `docs/`**（那是产品角色的）。**唯一例外**：更新 `docs/tasks/` 工单的 `status` 与「完成说明」。

## 提交规范

- 每完成一个工单提交一次：`feat(T0X): ...` 或 `chore: ...`。
- 提交信息末尾署名（用你自己的身份，例如 `Co-Authored-By: Codex <...>`）。
- 不做破坏性 git 操作；不丢弃他人改动；改动前先 `git status`。

## 验收

以工单「验收标准」+ `docs/spec/spec.md` 第 7 节为准。产品角色会按此 Review。

## 设计

设计由设计角色（另一个 Claude 对话）以文档形式给出，放在 `design/`。涉及样式落地的工单会指向对应设计文档；没有设计文档时，先做朴素可用的样式，不要自创复杂视觉。

## 角色边界速查

| 产品/架构 | Claude（产品对话） | `docs/`、工单 |

| 设计 | Claude（设计对话） | `design/` |
